---
title: Diferenças inventariais
sidebar_position: 2
---

O procedimento Diferenças inventariais é constituído na seguinte maneira:

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Procurar | Botão para aplicar os filtros de pesquisa dentro do banco de dados de acordo com os filtros inseridos. |
| Criação registração | Refere-se ao procedimento para o alinhamento dos estoques. |

ÁREA DE FILTRO: a área de filtro é dedicada à hospedagem na lista de possíveis tipos de dados através do qual é possível fazer uma seleção e uma pesquisa sobre o total dos artigos inventariais inseridos. Os filtros são em condição 'E' e é possível especificar mais critérios de filtro contemporâneos. Neste caso é possível filtrar por depósito e por artigo (classe, código artigo, descrição).

GRELHA DE RESULTADO: a grelha de resultado é uma lista dos artigos inventariais que correspondem aos dados de filtro especificados acima. O usuário, depois de especificar os valores pelos quais deseja obter uma pesquisa dos artigos inventariais, pressionando o botão [Pesquisa] presente na relativa Ribbon, resultará na grelha a lista dos artigos inventariais que quiser. A grelha de resultado apresenta uma série de informaç­ões sobre os artigos inventariados quais:

**Artigo** (classe, código artigo, matrícula);

**Eventual variante**;

**Opção**;

**Depósito**;

**Localização**;

**Encomenda de venda**;

**Cliente/Fornecedor**;

**Eventual lote**;

**Quantidade lógica a data elaboração o a data cálculo estoque**;

**Quantidade física relevada**;

**Diferença entre a quantidade lógica e a quantidade física**;

**Causal utilizada para o alinhamento** (para entender quais causais podem ser escolhidas se envia para a documentação relativa as tabelas da área logística, causais de depósito).

O procedimento permite escolher a data de elaboração enquanto procura em automático o último inventário não fechado. É possível efetuar a alinhamento dos estoques numa certa data marcando o elemento Calcular diferenças inventariais na data estoque e inserindo no campo Data cálculo estoque a data desejada. É também possível estabelecer a data da registração de depósito.
