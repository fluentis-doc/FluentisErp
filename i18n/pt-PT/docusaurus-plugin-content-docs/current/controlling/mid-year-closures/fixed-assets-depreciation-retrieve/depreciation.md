Na abertura da máscara precisa estabelecer obrigatoriamente:

- o **Encerramento de referência** (pela qual se lê a gama das datas de competência econômico);

- a **Data de registração** do movimento nas registrações extra-contábeis;

- a **Causal** a utilizar (aqui também é necessário que a causal prevê a gestão ,dos CdC/Encomendas para valorizá-las na registração).

O flag **Cada registração** para criar uma única registração extra-contábil com dentro as várias subcontas de amortização/fundo.

Lançando a procura dos dados, o programa vai valorizar a grelha da máscara com os dados das amortizações a calcular, proporcionadas com o número de dias de competência do encerramento. Em particular, para cada ativo fixo vai ser verificado se a amortização do ano (do ano estabelecido como parâmetro do encerramento) já está presente na sua ficha e esse valor vai ser reproporcionado no número de dias: se não estar presente, vai ser simulado um cálculo (de acordo com os parâmetros da categoria de pertença e dos restantes a disposição) e depois, esse valor vai ser proporcionado no número de dias de competência.

Lembra-se de selecionar todas as linhas presentes na grelha para valorizá-las no encerramento.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efeituar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Procurar | Executa a procura das retomadas das amortizações a registrar no encerramento infra-anual. |
| Amortização | Executa a relevação extra-contábil das retomadas das amortizações selecionadas. |
| Visualizar os ativos fixos | Visualiza o registro do ativo fixo a qual a retomada foi selecionada na grelha. |
