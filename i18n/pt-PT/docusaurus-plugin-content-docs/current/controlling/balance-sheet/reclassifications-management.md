title: Gestão reclassificações
sidebar_position: 8
---

Na inserção de uma nova reclassificação, na abertura da máscara é necessário estabelecer o código único de 5 caracteres alfanuméricos e a descrição da reclassificação, inserir o modelo de reclassificação e chamar de novo os dados do encerramentos das contas infra-anuais a utilizar. Se foi estabelecido o flag de 'Consolidado' vai ser possível chamar de novo o código da consolidação intersocietária a gerir no reclassificado.

No cabeçalho da máscara são presentes, também, um campo de ligação com as encomendas de venda, um para os centros de custo e um último para os centros de lucro: uma vez estabelecida uma destas, o gestional vai recalcular toda a estrutura do reclassificado retomando os dados das registrações infra-anuais associadas com o filtro estabelecido.

Ultimo campo do cabeçalho da reclassificação é denominado** Encerramento de ref. para o cálculo em aumento**: se refere ao valor inicial na qual o programa vai calcular as diferenças para valorizar os detalhes com o tipo salto 'aumento' (ver a documentação de detalhe dos tipos de nível subcontas e encomendas).

Abaixo desta seção, no lado esquerdo da máscara, vai ser reproposta a estrutura do modelo de reclassificação valorizada pelos dados do encerramento/consolidado estabelecido. Selecionado dessa parte um nível de tipo diferente da 'soma dos filhos' ou 'expressão' o gestional vai mostrar no lado direito duas grelhas: naquela superior vai-se visualizar a lista dos saldos do encerramento (consolidado) coerente com as impostações do mesmo nível, naquela inferior a lista dos dados que valorizaram efetivamente o nível. Através do botão presentes entre as duas grelhas vai ser possível fazer uma modifica diretamente no reclassificado sem modificar o modelo de referência. É permitido também a intervenção manual nos dados presentes na grelha abaixo, com também a possibilidade de inserir novas linhas manuais.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efeituar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Salvar | Salva a reclassificação em uso. |
| Inserção | Posiciona o cursor na inserção de um novo detalhe na linha das subcontas/CdC/CdL/encomendas do nó selecionado. |
| Procurar o nó sucessivo | Procurar o agrupamento nos grupos sucessivos aquele selecionado. |
| Procurar o nó precedente | Procurar o agrupamento nos grupos precedentes aquele selecionado. |
| Explosão do nó | Abre o detalhe do nó selecionado. |
| Implosão do nó | Fecha o detalhe do nó selecionado. |
| Aumento da tabulação | Aumenta a distância visualizada entre os níveis dos grupos na árvore. |
| Diminuição da tabulação | Diminui a distância visualizada entre os níveis dos grupos na árvore. |
| Procurar | Executa a procura na lista compressiva das subcontas do encerramento utilizáveis para o nó selecionado. |
| Procurar sub | Executa a procura na lista das subcontas inseridas no grupo selecionado. |
| Recalcular | Recalcula os valores da reclassificação. |
