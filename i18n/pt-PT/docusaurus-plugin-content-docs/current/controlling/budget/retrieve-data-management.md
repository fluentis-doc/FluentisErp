title: Gestão retomada dos dados
sidebar_position: 6
---

A partir desta máscara se atribuam os dados dos encerramentos infra-anuais aos vários modelos de reclassificação: lembrar que é necessário que o encerramento tenha a ligação com o período orçamento e com o detalhe período. A criação de uma nova retomada dos dados vai propor no cabeçalho o último encerramento criado disponível para a retomada dos dados: uma vez estabelecida uma descrição, o programa vai mostrar nas grelhas abaixo a lista dos períodos orçamento coerentes com a gama de datas inseridas no encerramento e o detalhe dos suborçamento pelas quais é possível proceder na retomada dos dados. Uma vez selecionados todos esses vai precisar apertar o botão 'Criação' da retomada dos dados para atribuir os valores relativos. Precisa lembrar que se houver suborçamento no modelo, vai precisar que seja criada uma reclassificação que valorize os cálculos do modelo para o consuntivo a orçamento.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efeituar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Salvar o registro do orçamento | Salva a retomada dos dados. |
| Criação | Executa o cálculo da retomada dos dados. |
| Restaurar | Anula o cálculo da retomada dos dados. |
| Visualizar o suborçamento | Visualiza os detalhes do suborçamento selecionado. |
