Nos Resumos são apresentadas as informações principais do inteiro documento.

**Descontos finais**: podem ser inseridos só os descontos expressos em porcentagem. Os descontos inseridos nesta seção vão ser calculados em cada artigo do documento;

**Resumo Imposto**: é proposto o resumo imposto do documento, para cada código imposto;

**Despesas**: são propostas as despesas inseridas no registro cliente, no tab ‘Despesas/Descontos'. Além disso há a possibilidade de inserir despesas que fazer referência só ao documento em uso;

**Notas finais**: é um campo livre que pode ser compilado pelo usuário com a ‘Ajuda codificadas';

**Totais documento**

**Quantia bruta artigos**: representa a soma dos valores de todos os artigos;

**Total descontos aplicados**: representa o valor total dos descontos aplicados sem os descontos finais;

**Quantia líquida artigos**: quantia bruta artigos – total descontos aplicados;

**Total descontos finais**: representa o valor dos descontos finais;

**Despesas não documentadas**: representa o valor das despesas inseridas como artigos de tipo ‘Despesas;

**Despesas documentadas**: representa o valor das despesas inseridas na grelha Despesas;

**Tributável**: quantia líquida artigos – descontos finais – total despesas não documentadas + total despesas documentadas;

**Imposto**: representa a soma dos valores contidos nos resumos imposto;

**Total**: tributável + imposto.

**Transporte**

**Volume**: é proposto o volume cumulativo, resultado da soma dos volumes dos artigos (o valor vai se retomado pelo registro artigo pesos/dimensões), quando a unidade de medida do volume dos artigos corresponde aquela inserida nos parâmetros da fatura. São pegos em consideração só os artigos que tem este campo compilado;

**Peso líquido**: é proposto o peso líquido cumulativo, resultado da soma dos pesos dos artigos (o valor é retomado pelo registro artigo pesos/dimensões), quando a unidade de medida do peso dos artigos corresponde aquela inserida nos parâmetros da fatura. São pegos em consideração só os artigos que tem o mesmo campo compilado;

**Pacotes**: é proposto o número de pacotes, de acordo com os dados inseridos no registro artigos, no tab Pesos/Dimensões, nos campos ‘Artigos presentes num pacote' ou ‘Pacotes para formar o artigo'. São pegos em consideração só os artigos que tem este campo compilado.

**Transporte pago por**: é proposto automaticamente o ‘transporte pago por' presente no registro do cliente.
