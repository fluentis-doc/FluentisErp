O procedimento 'Abertura automática contas' é uma das opções que se pode utilizar inicialmente para relevar os saldos de começo atividade: a partir do primeiro encerramento de balanço, as sucessivas aberturas das contas vão ser geridas diretamente pelo procedimento de encerramento. O interesse para a máscara, portanto, é relativamente baixo. Com este procedimento se registram unicamente os saldos contábeis de abertura, não a reabertura das partidas contábeis relativas.

O procedimento prevê de criar uma nova abertura, que vamos explicar campo por campo as funcionalidade:

**Data abertura**: é a data da operação, que não tem nenhuma ligação com a data de registração da operação que se vai a indicar na base da janela;

**Número abertura**: é um número progressivo gerido automaticamente;

**Descrição**: permite ao usuário identificar a operação, assim como as anotações;

**Subconta** e **Tipo conta**: são filtros de pesquisa dentro da lista do plano das contas;

**Data registração** e **Data competência**: movendo-nos na base da máscara vai precisar estabelecer esses dois campos para atribuir na registração de abertura;

**Número**: vai ser atribuído uma vez operada a contabilização;

Causal de abertura das contas e Descrição do movimento: a atribuir para o livro jornal.

Uma vez estabelecidos esses dados obrigatórios vai ser possível salvar as impostações e ativar toda a lista das subcontas contábeis: a fase sucessiva vai ser portanto aquela de operar a memorização subconta por subconta do saldo débito ou crédito da reabertura das contas, com a eventual quantia de reabertura na moeda se for necessário. Se uma vez  o total débito e o total crédito correspondam vai ser possível proceder com a relevação na contabilidade da retomada dos saldos: o usuário vai poder portanto reabrir a mesma abertura das contas para completar todos os dados (a operação pode ser longa e se realizar em mais dias) antes de proceder, ou se pode criar tantas aberturas contas diferentes, talvez diferenciando uma para o ativo, a outra para o passivo, ou para os clientes/fornecedores, etc.

Na máscara não está previsto um procedimento de redução: pode-se simplesmente cancelar a abertura e o aplicativo vai pedir confirmação do cancelamento da relativa registração de contabilidade.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Salvar | Botão para salvar o cabeçalho da abertura das contas, ativável depois de ter inserido a descrição e a causal contábil. |
| Procurar | Botão para procurar a lista das contas, que se quer inserida na abertura, através os campos da subconta e do tipo conta. |
| Criar registração abertura | Botão para criar a registração de abertura das contas. |
| Abrir a registração para a abertura contas | Botão que abre a registração de abertura. |
