title: Tab Agrupamento partidas
sidebar_position: 9
---

A gestão dos 'Agrupamentos partidas', uma vez inserida, é uma gestão incompatível com a normal gestão de partidas e pagamentos: a escolha entre uma gestão e a outro, portanto, é efetuada inicialmente no ato da impostação das causais de contabilidade geral com bem clara as características dessa gestão.

Com uma causal de tipo Imposto que tem o flag 'Agrupamento partidas' vai-se ativar automaticamente também a ficha de abertura das partidas: vai ser portanto executada uma abertura das partidas em relação as impostações do registro cliente/fornecedor. Dentro da ficha 'Agrupamentos partidas' o usuário pode proceder a criar, para essas partidas, um ou mais novos grupos de partidas (que vão pegar número e data do número e data documento da partida que o cria). A gestão dessas operações se efetua utilizando o menu contextual que aparece apertando o botão direito do mouse partida por partida: pode-se selecionar mais partidas/grupos (opção 'Mark'), criar novos grupos ('Criar novos grupos'), de-agrupar ('De-agrupar'), criar bonificações para fechar um grupo ('Bonificações'), ou visualizar os detalhes de cada partida ('Detalhes') e subdividir a partida inserida num grupo ('subdividir partida...'). A visualização dos grupos é efetuada para a subconta selecionada na grelha em alto.

No caso de registrações de cobrança/pagamento a operação do usuário é completamente diferente daquele de um encerramento partidas normal: nesse caso, o usuário vai começar a registração da ficha da mesma registração, relevando diretamente um valor do pagamento/cobrança, pela qual o aplicativo vai criar uma partida crédito/débito que o usuário vai proceder a atribuir manualmente ( se quer) um grupo da ficha 'Agrupamentos partidas' Operativamente o aplicativo vai criar, em cada uma dessas partidas, os relativos pagamentos recíprocos (e é por isso que a leitura de um plano de trabalho 'Standard' não vai ser mais claro e vai ter que utilizar só os relatórios e as máscaras de visualização/gestão dos agrupamentos para ter uma visão clara dos livros-razões). Quando o saldo do grupo, por efeito das várias partidas atribuídas, vai a 0, o grupo vai passar automaticamente para o estado 'Fechado.
