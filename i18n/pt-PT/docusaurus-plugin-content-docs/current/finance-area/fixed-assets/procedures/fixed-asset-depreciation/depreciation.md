Antes de tudo é de fundamental importância gerir os flag Operações incrementais: sendo o procedimento de amortizaçao gerida com logicas diferentes, se a empresa utiliza a opção incremental para as suas categorias vai precisar importar esse flag com o objetivo de individuar os dados. Na base da máscara, ao contrário, se estabelece o ano de referência do cálculo: normalmente vai precisar modificar com o objetivo de inserir o ano precedente ao atual. Não é permitido o cálculo das amortizações do ano X quando não foram contabilizadas as amortizações do ano X-1.

Uma vez efetuada a pesquisa dos dados vamos ver que é possível, linha por linha, ir a forçar o número de dias de aplicação assim como a porcentagem de amortização: se aconselha, na verdade, de efetuar o cálculo desses casos particulares diretamente dentro de cada ficha ativo fixo ou de modificar diretamente a tabela das categorias ativos fixos com o objetivo de efetuar um cálculo rápido com uma seleção total das linhas. Uma opção alternativa a utilizar, se quer utilizar porcentagens diferentes entre os vários ativos fixo, pode ser a definição do parâmetro do módulo ativos fixos nominado 'Porcentagem em cada ativo fixo', que permite definir no cabeçalho uma porcentagem de amortização a utilizar.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Procurar | Procura a lista das amortizações a calcular. |
| Amortização | Executa o cálculo para as linhas selecionadas. |
| Visualizar ativo fixo | Abre em visualização o registro do ativo fixo cuja amortização foi selecionada na grelha. |
