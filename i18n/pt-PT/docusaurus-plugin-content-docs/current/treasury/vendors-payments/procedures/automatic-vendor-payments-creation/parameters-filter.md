title: Filtro parâmetros
sidebar_position: 1
---

A máscara apresenta na primeira ficha uma série de filtros de pesquisa: precisa avisar que é possível filtrar também por códigos ABI/CAB dos apoios bancários associados com a partida contábil. São exclusos de default os pagamentos das partidas beneficiários, pelas quais a operação é efetuada dentro do módulo beneficiários se deseja obter a certificação das retenções. O filtro para a subconta banco permite de gerir a criação da lista em relação aquilo que foi canalizado nos vários bancos através das impostações de registro e do procedimento de gestão apoios bancários do mesmo módulo.

Na base do primeiro tabulador estão presentes alguns campos de totalização das linhas selecionadas.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Procurar partidas | Botão para ativar a pesquisa das partidas depois de ter especificado os filtros pelas quais obter o dado. |
| Iniciar a criação do pagamento das partidas | Botão ativo no tab Dados pagamento fornecedor. |

ÁREA DE FILTRO: a área de filtro é dedicada à hospedagem na lista de possíveis tipos de dados através do qual é possível fazer uma seleção. Os filtros são em condição 'E' e é possível especificar mais critérios de filtro contemporâneos.

GRELHA DE RESULTADO: a grelha de resultado é uma lista das partidas que correspondem aos dados de filtro especificados acima. O usuário, depois de especificar os valores pelos quais deseja obter uma pesquisa das partidas, pressionando o botão [Pesquisa] presente na relativa Ribbon, resultará na grelha a lista das partidas.
