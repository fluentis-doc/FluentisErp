RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Procurar pagamentos | Botão para a pesquisa dentro do banco de dados dos pagamentos fornecedores, depois de ter estabelecido, opcionalmente, os adequados campos de filtro da área Filtro. |
| Novo pagamento fornecedor | Botão para inserir um novo pagamento fornecedor. |
| Modificar os pagamentos selecionados | Botão para modificar um pagamento fornecedor precedentemente inserido, ativável depois de ter selecionado um pagamento. |
| Visualizar os pagamentos selecionados | Botão para visualizar um pagamento fornecedor precedentemente inserido, ativável depois de ter selecionado um pagamento. |
| Cancelar os pagamentos selecionados | Botão para cancelar um pagamento fornecedor precedentemente inserido, ativável depois de ter selecionado um pagamento. |

 

**Para as informações, relativas as Imprensas e aos Procedimentos que seguem, se envia aos respectivos elementos de Menu**



| RibbonFunction | RibbonMeaning |
| --- | --- |
|  [Imprimir pagamentos fornecedores](/docs/treasury/vendors-payments/reports)  | Refere-se ao procedimento de imprensa relativa aos Pagamentos fornecedores. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
|  [Bloqueio partidas passivas](/docs/treasury/vendors-payments/procedures/block-passive-maturity-values)  | Botão para proceder com a gestão massiva dos bloqueios das partidas passivas. |
|  [Criação automática pagamento fornecedores](/docs/treasury/vendors-payments/procedures/automatic-vendor-payments-creation/parameters-filter)  | Botão para a criação automática de um pagamento fornecedor. |
|  [Gestão apoios pagamentos](/docs/treasury/vendors-payments/procedures/payments-support-management)  | Botão para gerir os apoios bancários dos pagamentos. |
|  [Contabilização](/docs/finance-area/professional-men/accounting/payments-accounting/filter)  | Botão para proceder com a contabilização dos pagamentos fornecedor. |

ÁREA DE FILTRO: a área de filtro é dedicada à hospedagem na lista de possíveis tipos de dados através do qual é possível fazer uma seleção. Os filtros são em condição 'E' e é possível especificar mais critérios de filtro contemporâneos.

GRELHA DE RESULTADO: a grelha de resultado é uma lista dos pagamentos fornecedor que correspondem aos dados de filtro especificados acima. O usuário, depois de especificar os valores pelos quais deseja obter uma pesquisa dos pagamentos fornecedor, pressionando o botão [Pesquisa] presente na relativa Ribbon, resultará na grelha a lista registrações que quiser.
