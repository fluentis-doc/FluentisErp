A parte superior da máscara é dedicada a impostação dos filtros de pesquisa. Merecem uma menção particular os flag de pesquisa em relação a impostação do flag de agrupamento vencimentos dos registros clientes. São visíveis nesta seção as partidas contábeis associadas com os tipos pagamento de natura recebimento bancário ou nota promissória.

Na base da máscara precisa estabelecer a data de emissão dos efeitos enquanto o tipo efeito pode ser atribuído diretamente dentro da tabela dos tipos pagamento.

A seção **Contabilização** pode ser estabelecida de default dentro dos parâmetros do módulo Carteira efeitos. Se trata de definir a causal de contabilidade, as datas de referência para a registração e os eventuais flag de agrupamento e de registração provisória.

O flag **Agrupamento por conta efeito** significa executar uma registração do tipo 'conta efeito' a vários, onde vão estar presentes tantas linhas quantas são os efeitos a contabilizar.

O flag **Gestão provisória** vai a gerar as registrações contábeis de tipo provisório.

A **Gestão da aquisição efeitos das partidas** é alternativa a aquisição das faturas.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Procurar | Procura a lista das partidas pelas quais criar os efeitos. |
| Aquisição | Executa a criação dos efeitos para as partidas selecionadas. |
