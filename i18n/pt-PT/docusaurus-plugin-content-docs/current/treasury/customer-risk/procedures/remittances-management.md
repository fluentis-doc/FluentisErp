A máscara apresenta o detalhe das registrações de cobrança relevadas do cliente, dentro das gamas de datas e dos filtros inseridos no cabeçalho. O cálculo prevê uma contagem dos números associados com as variações de cobrança e dos números associados com os dias de variação do vencimento (da linha precedente) com o objetivo de calcular uma moeda média de cobrança e uma moeda média de vencimento (no caso de efeitos a data moeda foi estabelecida igual a data de vencimento, independentemente da data de contabilização do efeito). Da diferença desses dois valores se obtém um número médio de dias de atraso entre o vencimento e a moeda: esse número pode ser memorizado (através do botão 'Atualizar dias atraso') no registro no campo 'Dias médios atraso', campo utilizado nas simulações de fluxo de caixa para obter a data de vencimento prevista do fluxo financeiro.

A máscara prevê uma imprensa do detalhe do cálculo.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Procurar | Executa a pesquisa das registrações para o controle das remessas. |
| Atualizar os dias em atraso | O botão executa a atualização dos dias médios de atraso no registro do cliente nas análises. |
| Pré-visualização | Executa a pré-visualização de imprensa dos dados das remessas visualizadas.  |
| Imprimir | Executa a imprensa dos dados das remessas visualizadas. |
