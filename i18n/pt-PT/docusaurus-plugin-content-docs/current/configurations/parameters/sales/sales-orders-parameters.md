Os parâmetros das ordens clientes permitem a impostação de base para gerir corretamente e de acordo com os específicos pedidos de cada sociedade.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Restaurar os parâmetros | Permite restaurar os parâmetros nos valores iniciais. |

#1.1 Tab geral

| RibbonFunction | RibbonMeaning |
| --- | --- |
| Propor as comissões para todos os artigos  | Não mais gerido. |
| Gestão dupla unidade de medida | Avisa o fato que é gerida a unidade de medida alternativa na ordem cliente. |
| Proposta automática UM alternativa | Se ativado, é gerida a unidade de medida alternativa na ordem cliente. |
| Cobre os buracos de numeração | Se ativado, vão ser recuperados os números disponíveis. |
| Código oper. obrigatória | Se ativado, a específica do operador no cabeçalho ordem cliente é obrigatória. |
| Controlle sequência data e número documento | Se ativado, vai ser verificada a sequencialidade data/número para as ordens clientes. |
| Recuperar só no dia | Se ativado, vão ser recuperados os números disponíveis só pelo dia de hoje. |
| Recuperação automática | Avisa o fato que vai ser proposta em automático o primeiro número disponível. |
| Inserção automática variantes nos atributos variantes | Se ativado, as variantes dos atributos vão ser inseridas em automático no registro.  |
| Explodir a estrutura de produtos | Se ativado, na inserção de um artigo vai ser substituído pelo primeiro nível da sua estrutura de produtos, se existe. |
| Controlar a disponibilidade | Se ativado, vai ser verificada a disponibilidade do artigo ordem cliente. |
| Excluir as ordens clientes | Se ativado, a utilização das ordens clientes não vai ser considerada no cálculo da disponibilidade do artigo ordem cliente. |
| Dispon. obrigatória | Se ativado, a disponibilidade do artigo ordem cliente vai ser obrigatória, quer dizer, a quantidade não vai poder ser maior da disponibilidade. |
| Visualizar a disp. | Se ativado, vai ser visualizada a disponibilidade a nível de linha artigo quando se insere a quantidade. |
| Empreiteiro | Se ativo, os documentos de empreiteiro vão ser considerados no cálculo da disponibilidade do artigo ordem cliente. |
| Compras | Se ativado, os documentos de compra vão ser considerados no cálculo da disponibilidade do artigo ordem cliente. |
| Vendas | Se ativado, os documentos de venda vão ser considerados no cálculo da disponibilidade do artigo ordem cliente. |
| Prod. planejada | Se ativado, os documentos de necessidade materiais e encomenda de produção vão ser considerados no cálculo da disponibilidade do artigo ordem cliente. |
| Prod. libertada | Se ativado, as ordens de produção vão ser consideradas no cálculo da disponibilidade do artigo ordem cliente. |
| Depósito | Se ativado, as registrações do depósito vão ser condideradas no cálculo da disponibilidade do artigo ordem cliente. |
| Procurar o preço artigo em todos os catálogos predefinidos | Se ativado, o artigo inserido na linha ordem cliente vai ser procurado também nos catálogos especificados no registro cliente, além do catálogo de default. |
| Uso BarCode | Se ativado, as colunas Artigo cliente e Descrição artigo cliente da grelha Artigos são visíveis e geridas. |
| Uso artigo cliente | Se ativado, as colunas Artigo cliente e Descrição artigo cliente da grelha Artigos são visíveis e geridas. |
| Gestão tamanho e cor | Se ativado, as colunas Lote da grelha Variantes são geridas. |
| Visualizar as dimensões | Se ativo, as colunas dimensões (altura, largura e profundidade) da grelha Artigos são geridas. |
| Consentir descontos para os artigos de brinde | Se ativado, os artigos brinde geram os descontos. |
| Controlar a integridade | Se ativado, se verifica a integridade dos dados da ordem cliente. |
| Obrigação relação pacotes em quant. | Se ativado, vai ser verificada a quantidade da linha ordem cliente relativamente ao número dos pacotes do registro artigo. |
| Bloquear a inserção documento nos dias de feriado | Se ativado, impede a utilização dos dias de feriado. |
| Propor os lotes econômicos | Se ativado, é proposto o lote econômico do registro artigo para a quantidade da linha ordem cliente. |
| Ligação ord.cl./prod. | Se ativado, a quantidade da linha ordem cliente vai continuar igual a quantidade da encomenda de produção, cujo é associada. |
| Unir os atributos | Se ativado, vão ser inseridas como atributos a linha ordem cliente, seja os atributos do cliente que aqueles do registro artigo. |
| Bloquear os atributos | Se ativado, a grelha Atributos do artigo ordem cliente vai ser bloqueada. |

#1.3 Tab configurador

| RibbonFunction | RibbonMeaning |
| --- | --- |
| Configurador | Específica o endereço HTTP configurador. |
| Ordens clientes  | Específica o endereço HTTP WSOC. |
| Versãoersione | Específica o tipo de configurados utilizado: Configurador ou Visual trailor. |

#1.4 Tab execução

| RibbonFunction | RibbonMeaning |
| --- | --- |
| Pagamentos | Específica o valor utilizado para o pagamento da ordem cliente no caso de execução múltipla encomenda de venda: primeira ordem, registro ou seleção manual. |
| Destinatário | Específica o valor utilizado para o destinatário da ordem cliente no caso de execução múltipla encomenda de venda: primeira ordem, registro ou seleção manual. |
| Execução da pesquisa das ordens e Agrupamento por tipo e solução de pagamento | Específica o fato que as ordens clientes vão ser agrupadas por tipo e solução de pagamento na execução direta nas notas de entrega ou nas faturas. |
| Execução da pesquisa ordens e Agrupamento para o destino | Específica o fato que as ordens clientes vão ser agrupadas pelo destinatário e pelo destino na execução direta nas notas de entrega ou nas faturas. |

#1.5 Tab analíticaEspecífica a prioridade para a pesquisa CdC/CdL: Tipo faturado, Registro, Registro artigos ou Depósito.
