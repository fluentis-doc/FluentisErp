Os parâmetros de pedidos de compra permitem a impostação de base para gerir corretamente e de acordo com os específicos pedidos de cada sociedade.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Restaurar os parâmetros de default | Permite restaurar os dados iniciais. |

#1.1 Tab geral

| RibbonFunction | RibbonMeaning |
| --- | --- |
| Custo zero em ausência de catálogos | O preço é procurado nos catálogos; se não é encontrado e é ativo este flag na linha de pedida se coloca o preço 0. Se não é ativo e não é ativo nem o outro parâmetro e não encontra o preço no catálogo, procurar o custo último no registro artigo e se não o encontra, colocar 0. |
| Custo último em ausência de catálogos | Quando é ativo, se não encontra o preço no catálogo e não encontra o custo último no registro artigos se pega o máximo entre os preços presentes no pedido de compra do mesmo fornecedor e artigo. |

#1.2 Tab analíticaEstes parâmetros indicam a prioridade de pesquisa dos centros de custo ou de lucro para atribuí-los aos artigos do pedido de compra. Quando o flag Recalcular for estabelecido, os centros de custo ou de lucro vão ser inseridos/atualizados nos artigos através a prioridade escolhida.
