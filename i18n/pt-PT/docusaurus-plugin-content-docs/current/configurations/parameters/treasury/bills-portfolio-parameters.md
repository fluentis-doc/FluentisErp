RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Salvar | Botão para salvar as impostações. |
| Anular as impostações | Botão para anular as impostaç­ões. |

 

 

Os parâmetros do módulo Carteira efeitos se encontram na [Home > Parâmetros > Tesouraria].

Nos parâmetros do módulo Carteira efeitos se podem preestabelecer os elementos de default a propor nos procedimentos do módulo. Em particular:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Contabilização automática | Permite predefinir a contabilização efeitos diretamente no momento da aquisição efeitos pelas faturas ou partidas. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Causal de contabilidade | Permite definir a causal contábil de default a propor no procedimento homônimo. |
| Agrupar por conta efeito | Permite definir o parâmetros de agrupamento efeito pelas registrações efetuadas com tal procedimento. |
| Registrações provisórias | Permite definir o estado de registração provisória pelas registrações efetuadas com tal procedimento. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Causal de contabilidade | Permite definir a causal contábil de default a propor no procedimento homônimo. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Causal de contabilidade | Permite definir a causal contábil de default a propor no procedimento homônimo. |
| Contabilizar na data de maturação | A primeira opção vai a romper a registração da lista nas várias datas em vencimento dos efeitos com a adição dos dias do banco estabelecidos no registro banco pela causal estabelecida. |
| Contabilizzar na data em vencimento com a data moeda banco = data de maturação | A segunda opção, ao contrário, vai registrar os vários efeitos nas várias datas de vencimento mas com a data moeda banco rectificada de acordo com os dias banco estabelecidos no registro banco pela causal estabelecida. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Causal de contabilidade | Permite definir a causal contábil de default a propor no procedimento homônimo. |
| Despesas comissões | Permite definir a subconta das despesas de comissões para as registrações efetuadas com tal procedimento. |
| Taxas passivas | Permite definir a subconta das taxas passivas para as registrações efetuadas com tal procedimento. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Agrupar quantias enferiores a | Este campo, se não é vazio e diferente de zero, determina o valor limite das partidas a agrupar. Se, por exemplo, se estabelece '1000', significa que vão ser agrupadas as partidas inferiores a esta cifra, portanto até 999,99. |
