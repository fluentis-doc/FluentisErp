RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Salvar | Botão para salvar as impostações. |

  

Os parâmetros do módulo Pagamentos a fornecedores se encontram no [Home > Parâmetros > Tesouraria].

Nos parâmetros do módulo Pagamentos fornecedores se podem preestabelecer os elementos de default a propor nos procedimentos do módulo. Em particular:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Percurso de criação dos traçados Recibo bancários | Impostação da Directory padrão de criação do traçado telemático para as transferências bancárias Itália. Se aconselha de criar uma Directory repartida de rede dentro do server, para poder aceder por todas as colocações do procedimento.  |
| Padrão ABI | Impostação do formado do arquivo telemático. Nesta caso se sugere o padrão ABI, que segue as impostações padrão de comunicação assim como definidas para esta entidade. |
| Record HR | Impostação do formado do arquivo telemático. |
| ABI Centro aplicativo | Impostação de um código ABI do eventual centro aplicativo que vai avançar a veicular as comunicações (normalmente é um campo não necessário). |
| Impor o banco de apoio presente no registro do fornecedor | É possível definir a utilização do banco de apoio relevada de acordo com o default do registro associado: se refere a sobrescrição do banco de apoio inserida na registração contábil com aquela de default da grelha superior do registro bancos. |
| Gestão ref. IBAN | Impostação da obrigação de gerir alguns códigos IBAN: neste caso o traçado não vai ser criado se faltam os IBAN de alguns pagamentos na lista. |
