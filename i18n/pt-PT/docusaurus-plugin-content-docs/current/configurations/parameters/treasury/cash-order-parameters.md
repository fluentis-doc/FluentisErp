RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Salvar | Botão para salvar as impostações. |





 Os parâmetros do módulo Recibos bancários se encontram na [Home > Parâmetros > Tesouraira].

Nos parâmetros relativos se podem preestabelecer os elementos de default a propor nos procedimentos do módulo. Em particular:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Percurso de criação dos traçados recibo bancário | Impostação da Directory padrão de criação do traçado telemático para os recinos  bancários. Se aconselha de criar uma Directory repartida de rede dentro de um server,  para poder aceder a todas as postações do procedimento. |
| Referências do cliente | É a razão social da sociedade ativa. |
| Descrição padrão do crédito | É possível codificar, aproveitando dos códigos entre parêntesis, o texto a inserir no arquivo qual descrição do documento de referência da emissão do recibo bancário. Se refere a um texto do tipo Ft. nº (2) do (3). |
| Inserir carácter de controle | Para alguns programas de Remote Banking precisa uma nova linha de fuga do arquivo, estabelecer este flag se necessário. |
| Sem referência para a importação não pagos | Em alguns programas de Remote Banking é previsto uma atualização automática dor registros de acordo com os dados inseridos no arquivo recibo bancário. O campo que se aproveita para o procedimento de importação não pagos pode causar uma contínua atualização dos registros que se pode evitar com esse flag (coisa que comporta a não usabilidade do procedimento de importação não pagos). |
| Gestão vencimento descrição | Com este flag a descrição do cliente inserida no arquivo vai ser composta como primeiro descrição + segunda descrição. |
