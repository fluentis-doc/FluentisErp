Seção **Detalhe atributos** são presentes em várias impostações:

**Causal contabilidade industrial**: se o tipo registração prevê a contemporânea gestão da geral, é possível indicar (o campo não é obrigatório) neste campo a causal de contabilidade geral a propor na registração industrial;

Seção **Ulteriores colunas visualizáveis na grelha das registrações contábeis**:

**Tipo Imposto**: visualiza a seleção do tipo de movimento Imposto nas linhas de movimento industrial;

**Moeda**: visualiza a seleção da moeda nas linhas de movimento industrial;

**Dados Imposto (Cód. tributável...)**: visualiza os dados Imposto nas linhas de movimento industrial;

**Datas competências Da/A**: visualiza as gamas de datas de competência das linhas de movimento industrial;

**Descrição subconta**: campo não em uso;

**Data moeda**: visualiza a data moeda nas linhas de movimento industrial;

**Descrição Imposto**: campo não em uso;

**Detalhe movimento**: visualiza um campo notas nas linhas de movimento industrial;

**Quantia na moeda**: visualiza a quantia moeda nas linhas de movimento industrial;

**Câmbio**: visualiza o câmbio nas linhas de movimento industrial;

**% indedutibilidade**: visualiza a porcentagem de indedutibilidade nas linhas de movimento industrial.

Seção **Tipo registração**:

**Registração a quantidade**: esta opção ativa a possibilidade de inserir unidade de medida, quantidade e preço unitário nas linhas de contabilidade industrial baseadas na causal em uso;

**Registração a valor**: com esta opção a registração industrial vai ter só os campos do valor débito/crédito do movimento.

Seção **Tipo registração a quantidade**, ativa só se com registração a quantidade:

**Artigos vendas**: permite associar a linha de movimento o artigo de depósito;

**Artigos compra**: permite associar a linha de movimento um artigo fornecedor;

**Sem artigos**: não prevê a associação de uma linha artigo.

Seção **Parâmetros de registração**:

**Bloquear o salvamento se tem desequilíbrio econômico geral**: com este flag não vai ser possível salvar a registração se não existe uma correspondência dos valores econômicos entre a seção industrial e geral;

**Bloquear o salvamento se tem desequilíbrio industrial**: com este flag não vai ser possível salvar uma registração industrial desequilibrada;

**Bloquear a inserção subcontas não econômicas**: com este flag não vai ser possível movimentar as subcontas industriais não econômicas.

Seção **Tipo registração a valor**:

**Centros de custo**: para as registrações a valor não associadas com as causais geral, com esse flag é possível habilitar a ficha de movimentação no centro de custo;

**Centros de lucro**: para as registrações a valor não associadas com as causais geral, com esse flag é possível habilitar a ficha de movimentação no centro de lucro;

**Encomendas**: para as registrações a valor não associadas com as causais geral, com esse flag é possível habilitar a ficha de movimentação na encomenda.

Seção **Opções registração**:

Se as causais não preveem uma movimentação geral, nesta seção é possível indicar se a registração, no salvamento, vai ser replicada para os períodos estabelecidos nos dois campos:

**Periodicidade**: tipo de período a utiliziar para a replicação:

**Número períodos**: número de duplicados a criar com a periodicidade indicada.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Novo atributo | Não gerido. |
| Cancelar atributos | Não gerido. |
| Salvar atributos | Não gerido. |
