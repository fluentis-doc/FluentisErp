A tabela 'Registros' se compõe de 4 seções:

**Plano das contas**: nesta seção precisa indicar, para as subcontas dos registros banco, os campos 'Banco' e 'RBN'. O primeiro indica o instituto de crédito de referência, enquanto o segundo o tipo de conta (conta ordinária, sbf, ...) associado com a mesma subconta: esses dois campos são visíveis e modificáveis também dentro do registro banco. Pelo que se concerne as subcontas de tipo cliente/fornecedor, é possível gerir a partir desta máscara também o campo da 'Derrogação', campo que pode se gerir também pela máscara principal do registro contatos, para cada linha de subconta;

**Fluxo plano de trabalho**: nesta seção se atribui para cada fluxo de fluxo de caixa (refere-se a documentação de base do homônimo módulo) o relativo código por DocFinance. Permite-se a atribuição do mesmo código em mais linhas de fluxo;

**Tipos pagamento**: nesta seção se indica, para cada tipo de pagamento, o relativo código de correspondência no DocFinance (é permitida a atribuição do mesmo código em mais linhas). É possível também indicar um código de derrogação do tipo de pagamento, que vai ser utilizado na exportação dos dados para o DocFinance;

**Sociedade**: para cada sociedade é obrigatório indicar o código correspondente para o DocFinance. As sociedades que não têm código vão ser exclusas dos procedimentos de exportação/importação dos dados para/do DocFinance.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Procurar | Com este botão se executa a pesquisa em todas as linhas da máscara, em relação aos filtros estabelecidos. |
