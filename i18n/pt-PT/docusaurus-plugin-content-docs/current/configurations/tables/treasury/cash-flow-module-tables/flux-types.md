A tabela prevê simplesmente a codificação das tipologias de fluxo do fluxo de caixa que a empresa quer utilizar para as simulações. São previstas as seguintes origem de dados:

**Partidas**: a origem dos dados são as partidas de contabilidade geral;

**Saldo contábil**: a partir desse fluxo se obtém as disponibilidades financeiras iniciais, em relação aos tipos de contas financeiras codificadas;

**Ordens clientes/ordens fornecedores**: ordens clientes das tipologias com o flag 'Fluxo de caixa' imprimidos e confirmados não executados;

**Faturas de compra/venda**: faturas imprimidas/controladas mas ainda não contabilizadas (não de tipo próforma);

**Extracontábil**: vencimentos manuais extracontábeis definidos no módulo Fluxo de caixa (ver o documento sucessivo);

**Beneficiários**: remunerações profissionais ainda não contabilizadas;

**Notas de entrega de compra/venda**: notas de entrega das tipologias com o flag 'Fluxo de caixa', imprimidos e confirmados, não valorizados;

**Ordens e retornos de empreiteiro**: ordens de empreiteiro a executar os retornos do empreiteiro ainda não faturados;

**PC**: pedidos de compra;

**Ordens planejadas**: ordens de produção com fases externas planejadas ou ordens de produção que preveem um pedido de compra.

Se aconselha codificar todas as tipologias previstas: a geração automática permite definir quais dessas considerar e quais não.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:
