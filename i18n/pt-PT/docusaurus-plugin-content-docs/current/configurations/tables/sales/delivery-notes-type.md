---
title: Tipo notas de entrega
sidebar_position: 13
---

O tipo notas de entrega é utilizado para determinar as propriedades de um CT-e.

RIBBON BAR: representa o menu do módulo em questão, ou seja, a área na qual efetuar algumas ações. A lista de possíveis funcionalidades é sucessivamente representada:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Procurar | Botão para efetuar a pesquisa de um tipo documento. |
| Inserção | Botão para inserir um novo tipo documento. |
| Cancelamento | Botão para cancelar o tipo documento. |
| Pré-visualização de imprensa | Botão para executar a pré-visualização de imprensa. |
| Imprimir | Botão para executar a imprensa em relação aos filtros estabelecidos e a configuração da impressora. |

**Tipo**: tipo CT-e. 

**Descrição**: descrição do tipo CT-e.

**Tipo Num./Descrição**: tipo numeração associada com o tipo CT-e. 

**Natura CT-e**: determina o tipo CT-e: entrega, retorno, com visão, retorno com visão. 

**Ordem**: se estabelecido, significa que o CT-e pode ser criado executando uma ordem cliente.

**Valores**: se estabelecido, na imprensa do CT-e podem ser imprimidos também os valores.

**Tipo fatura/Descrição**: tipo fatura utilizado para a criação das faturas do CT-e. 

**Caus. Transp**.: descrição da causal de transporte.

**Controle cliente**: na inserção da subconta de título da ordem é verificada que o tipo conta é obrigatoriamente cliente.

**Bloquear doc. impr.**: se estabelecido, não é possível modificar um CT-e já imprimido. 

**Crédito**: se estabelecido, o CT-e com este tipo retorna no controle do crédito.

**Excluir o bloqueio**: se estabelecido, significa que mesmo se supera o crédito, e o documento vai ser bloqueado, deixa mesmo assim inseri-lo e imprimi-lo.

**Fluxo de caixa**: se estabelecido, o CT-e com este calcula o cálculo do Fluxo de caixa.

**Depósito/Descrição**: depósito de utilização da mercadoria contida no CT-e desse tipo.

**Causal/Descrição**: causal de movimentação da mercadoria utilizada contida no CT-e desse tipo.

**Sem somar a quantidade**: se estabelecido, na execução das ordens deste tipo as quantidades executadas são reportadas no documento de execução para cada linha de execução.

**Agr.Venda Imposto**: se estabelecido, se procura para o artigo se existe o código Imposto agrícola, se não existe se pega o código Imposto normal.

**Controlar a retirada de descarr.**: se estabelecido, quando se cria uma nota de entrega da retirada descarregada, se ativa em automático o flag descarregado para a nota de entrega.

**Bloquear a quant. retirada**: se é estabelecido, bloqueia a quantidade e a quantidade alternativa na nota de entrega criada pela lista UdC.

**Permitir os lotes não disponíveis**: se estabelecido, permite também o descarregamento dos lotes não disponíveis.

**Gest.EDI**: se estabelecido, para o CT-e deste tipo é possível criar o arquivo EDI.

**Preço com imposto**: se estabelecido, os preços presentes no CT-e compreendam o Imposto.
