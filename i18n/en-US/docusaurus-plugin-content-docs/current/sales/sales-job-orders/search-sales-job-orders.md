---
title: Search Sales Job Orders
sidebar_position: 3
---

The Search Sales Job Orders procedure has been created in order to enable the user to search easily the job orders previously inserted with the task to view, to edit, to delete them or to insert a new job order, after having obtained the necessary user's rights for these operations.

Thanks to filters it is possible to view: a document (e.g. search by document number), more documents (e.g. search by date), documents inserted in a year (e.g. search by year), documents inserted by a certain vendor (e.g. search by account/detail account), documents according to the estimated delivery date or confirmation date.

RIBBON BAR: it represents the Form menu, that is the area in which it is possible to perform actions. The list of features is the following:



| Function | Meaning |
| --- | --- |
| Search Job Order | Button to search job orders within database. |
| New Job Order | Button to create a new job order. |
| Edit Job Order | Button to edit a previously inserted job order. |
| View Job Order | Button to view the previously inserted job orders. No modification are allowed. |
| Delete Job Order | Button to delete a previously inserted job order. |
| Execute Orders | The automatic procedure enables the user to execute sales orders from a job order. |
| Duplicate Job Order | Button to duplicate a job order. It enables the creation of a document copy, reporting all the conditions of the source document, but with additional current data. |
| Property | It enables the user to view the execution of a job order, together with job order items data and data of documents associated. |
| Conversion | Button for the conversion of a document prices to another currency. |
| Preview | Button to perform the print preview of a document, according to filter criteria, above mentioned (job order type, number, date, customer). |
| Print | Button to print document, according to filter criteria, above mentioned (job order type, number, date, customer). |

FILTER AREA: the filter area contains the list of possible data types thanks to which it is possible to perform a selection. Typically filters have always 'AND' condition and it is possible to specify more filter criteria at the same time.

RESULT GRID: the result grid represents the list of job orders which correspond to the filter data, above mentioned. After having specified values through which it is possible to obtain a records search, the user, by clicking on [Search ] button on the related Ribbon, will obtain the list of desired job orders on result grid.
