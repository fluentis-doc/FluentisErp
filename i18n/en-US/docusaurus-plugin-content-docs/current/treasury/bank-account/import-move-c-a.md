This mask enables the user to import current accounts movements from telematic file (standard abi) that can be taken from home banking.

First, in order to import, it is necessary to set the detail account at the basis of mask for which to import file, then clicking on 'Execute Import' button it will be possible to open an opening mask of the telematic file. At the end of this operation a new import line will appear on the grid: through a double click on the same one or through the 'View file' button it will be possible to view the origin text file of imported data.

The imported movements will be used in the reconciliation C/A procedure

The Import Move. C/A procedure consists as follows:

RIBBON BAR: it represents the Form menu, that is the area in which it is possible to perform actions on procedure in use. The list of features is the following:



| Function | Meaning |
| --- | --- |
| Search Import | Button to apply search filters within database of the performed import. |
| Execute Import | Through this button it is possible to select the telematic file and its import. |
| View File | Through this button it is possible to view the telematic file associated to the selected import. |
| Delete Import | It deletes the selected import. |

FILTER AREA: The filter area contains the list of the possible data types thanks to which it is possible to perform a selection, a research on the total of data inserted in system. Typically Filters have 'AND' condition and it is possible to specify more filter criteria at the same time.

RESULT GRID: the result grid represents the import list that belongs to the filter data above mentioned. After having specified values through which to obtain a search of inventory items the user, by clicking on the [Search] button in the related Ribbon, will obtain the list of desired inventory items in the result grid.Â
