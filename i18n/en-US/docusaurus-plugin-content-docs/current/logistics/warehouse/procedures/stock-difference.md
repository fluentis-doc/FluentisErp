The inventory differences procedure consists as follows:

RIBBON BAR: the ribbon bar represents the Form menu, that is the area on which it is possible to perform actions. The possible features list is the following:



| Function | Meaning |
| --- | --- |
| Search | Button to apply search filters within database according to the inserted filters. |
| Create Record | It refers to the procedure to align stocks. |

FILTER AREA: The filter area contains the list of the possible data types thanks to which it is possible to perform a selection, a research on the total of data inserted in system. Typically Filters have 'AND' condition and it is possible to specify more filter criteria at the same time. In this case it is possible to filter per warehouse and item (class, item code, description).

RESULT GRID: the result grid represents the inventory list that belong to the filter data above mentioned. After having specified values through which to obtain a search of inventory items the user, by clicking on the [Search] button in the related Ribbon, will obtain the list of desired inventory items in the result grid. The result grid shows a series of information about inventory items, such as:

**Item **(class, item code, code)

**Possible Variant**

**Option**

**Warehouse**

**Location**

**Sales Job Order**

**Customer/Vendor**

**Possible Lot**

**Logic quantity at the elaboration date or stock calculation date**

**Obtained Physical Quantity**

**Difference between the logic and physical quantity**

**Template used for the alignment** (in order to understand which are the choices it is recommended to view the documentation related to the logistic area tables, warehouse template).

The procedure enables the user to choose the elaboration date during the automatic search for the last not closed inventory. It is possible to perform the stocks alignment to a certain date marking off the Calculate inventory differences at stock date entry and inserting the desired date into the Stock Calculation Date field.
