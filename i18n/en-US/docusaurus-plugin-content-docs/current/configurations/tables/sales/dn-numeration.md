The DN numeration table enables the user to set different progressive numeration within the same year to be attributed to one or more DN types.

The ribbon bar represents the Form menu, that is the area in which it is possible to perform actions linked to the procedure in use. The possible features list is the following:Â 



| Function | Meaning |
| --- | --- |
| Search | Button to search a numeration type.  |
| Insert | Button to insert a new numeration type.  |
| Delete | Button to delete the numeration type. |
| Print Preview | Button to perform the print preview.  |
| Print | Button to print according to the filters set and to the printer configuration.  |



| Function | Meaning |
| --- | --- |
| Year   | Numeration year. |
| Type | Numeration type. |
| Description   | Numeration type description. |
| Progr. No. | Starting progressive number for DN numeration. |
| Sales Returns | If it is set, it indicates that the numeration type is used in order to number the returns, for this reason it does not verify the progressive number/date more but it is necessary to add its reference on note line, created within delivery note valorization. |
| Series | If it is inserted, it can be used for accounting in order to link both document number and series. |
| Fixed Number of Characters | If it exists, it can be used in order to create the series. |
