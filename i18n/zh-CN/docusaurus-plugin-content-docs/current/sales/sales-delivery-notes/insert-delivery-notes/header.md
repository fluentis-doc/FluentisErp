用户输入客户的时候，系统自动地表示“标头”表格的数据。这些数据按照客户登记中的设置。在登记中确定的表示的数据是：

**交货**： –表示客户登记中的数据，“交货”表格（参考的表格“交货”）

**运费**：表示客户登记的“交货”表格中的数据（参考的表格“运费”）

**包装**：表示客户登记的“交货”表格中的数据（参考的表格“包装”）

**国家**：表示客户登记的“一般数据”表格中的数据（参考的表格是“国家”）

**区域**：在表示客户登记的“交货”表格中的数据（参考的表格是“区域”）

**价目表**：表示属于客户登记的“价目表”表格中的数据，仅档有某个默认的提供（“销售价目表”的参考表格）

**从有效日期**：选择的价目表从本日期以后是有效的

**至有效日期**：选择的价目表至本日期之前是有效的

**语言**：表示客户登记属于的“一般数据”

**货币**：表示属于客户登记的“管理”表格中的数据（参考的表格是“货币”）

**货币转换**：如公司的货币相当于文档的货币的话价值为1，要不然它按照货币转换或者欧元的固定汇率

**货币日期**：计算汇率的日期，就是公司货币与客户货币的关系，按“货币转换”的设置

**初始的票据**：用户可以输入已在“编码票据”中的一些票据。首先用户得双击于“初始”字段，然后开“帮助编码票据”选择数据。通过鼠标右键，用户打开一个窗户添加很长的文章。如果字段包含价值，字段的背景颜色改变

**客户票据**：表示在属于客户登记的“行政”表格中的数据。通过鼠标右键用户打开一个窗户添加很长的文章。如果字段包含价值，字段的背景颜色改变

**目的地和接受者**：表示属于客户登记的交货表格中的数据，才有默认的目的地和接受者

组合框表示客户登记中的所有接受者和目的地

**用户**：输入创建文档的用户（参考的表格是“搜索职员”，在行政/个人的区域）。用户得选择文档的参数

**作业单**：将某个文档与某个销售作业单关系。文档状态：这种数据指文档状态的表格，用户得插入数据，系统的复制程序可以利用这些数据。EDI (电子数据交换) 是EDI 程序的交换日期

**货单的状态**：可以是已打印的、下载的、有价值的、取消的

**EDI**：传输文档的日期

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 保存 | 用于保存送货单/货单 |
|  [从订单的履行](/docs/sales/sales-delivery-notes/insert-delivery-notes/header-procedures/execution-from-order)  | 从销售订单生成货单 |
|  从仓库领货清单创建货单  | 用于利用在领货清单中的项目 |
| 从销售价目表中，交换项目 | 用于利用在销售价目表中的项目 |
| 从装箱单创建货单  | 用于利用在装箱单中的项目 |
| 帮助接受者/目的地 | 用于查看以及输入客户登记中的接受者/目的地 |
| 删除货单 | 用于删除已输入的货单 |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 文档经理 | 管理与在使用中的掩码有关的档案 |
