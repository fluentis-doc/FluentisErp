插入销售发票的程序需要：

**发票类型：**在[主页>表格>销售]预定的。按用户在“编号”表格中制定的编号，每个文档有自己的号码。年份和日期的字段是自动地填写的

插入客户的时候，按在客户登记中的数据系统自动地表示“头标”表格的所有数据。

登记表示的数据是：

**交货**：表示客户登记中的数据，“交货”表格（参考表格“交货”）

**运费**：表示在客户登记中的数据，“交货”表格中（参考表格是“运费”）

**包装**：表示客户登记中的数据，在“交货”表格中（参考的表格是“包装”）

**国家**：表示“共同数据”的数据，在客户登记中（参考的表格“国家”）

**区域**：表示属于客户登记的“交货”表格中的数据

**价目表**：表示属于客户登记的价目表表格中的数据，默认的提供是需要的条件（参考的表格是“销售价目表”）

**从有效日期**：从这个日期以后，选择的价目表有效

**至有效日期**：只这个日期，选择的价目表有效的

**语言**：表示客户登记的“共同数据”表格的数据

**货币**：表示属于客户登记的“财政”表格（参考的表格是货币）

**货币兑换**：如果公司的货币相当于文档的货币、价值为1，要不然系统按“货币兑换”或者“欧元固定汇率”的货币

**货币日期**：汇率的价值计算根据公司货币和客户货币的关系，按“货币兑换”的设置

**初始票据**：可以包括“编码票据”中的已插入的票据。首先用户双击于“初始”字段，然后开“帮助编码票据”来选择数据。另一个选择是首先击右鼠标按钮来开一个窗户因此插入票据的长文章。如果字段包括价值，字段的背景颜色改变

**登记的票据**：表示属于客户登记的“财政”表格中的数据。另一个选择是首先击右鼠标按钮来开一个窗户因此插入票据的长文章。如果字段包括价值，字段的背景颜色改变

**目的地和收件人**：表示属于客户登记的“交货”表格。用户：用于将创建文档的用户（参考的表格“搜索人员”）插入到财政/个人区域。在文档的参数内，用户可以设置本参数

**作业单**：将文档与销售作业单联系起来

**参考发票和类型**：首先用户得注销发票，然后将两个发票联系起来，插入本来发票号码、日期和类型

**记账记录**：表示会计记录的号码、日期以及凭证;文档状态：关于文档状态的表格，用户得插入数据。系统的复制程序可以利用这些数据来管理某个销售点。EDI指定从EDI程序文档转动的日期

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 保存 | 用于保存销售发票 |
| 从销售订单履行 | 用于利用从某个销售订单中创建发票的程序 |
| 从领料创建发票 | 用于利用在领料清单内的项目 |
| 从销售价目表创建发票 | 用于利用在销售价目表内的项目 |
| 从装箱单创建发票 | 用于利用装箱单内的项目 |
| 帮助收件人/目的地 | 用于查看以及插入客户登记中的收件人/目的地 |
| 删除发票 | 用户删除已插入的发票 |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 文档经理 | 用于按照已选择的筛选器和打印机的配置打印 |
