title: 重新分类的管理
sidebar_position: 8
---

插入重新分类的时候，用户得制定由5字/数字组成的唯一代码以及重新分类的说明、插入重新分类的模型、撤回得利用的期中结算的数据。掩码的标头包括与销售作业单的连接字段，一个关于成本中心，其他的关于利润中心：连接字段设置了之后，系统再计算重新分类的完全结构，撤回与筛选器联系的其中记录的数据。重新分类标头的最后字段叫**为增加计算的参考结算**：按照本初始价值，系统可以计算为制定包含“增加”余额类型的细节价值的差别（请按照关于明细账目的层次类型/作业单的细节文件）。在本部分下，在掩码的左边，系统表示重新分类模型的结构。结算/合并的数据制定本模型的价值。选择与“子级记合”或者“表达式”不同的层次，在右边系统表示两个网格：上面的包括与层次设置兼容的结算的余额列表（合并的）、下面的包括层次有价值的数据列表。通过两个网格中间的按钮用户可以直接到编辑重新分类，改变参考的模型不是需要的。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 保存 | 用于保存在使用中的重新分类 |
| 插入 | 新细节插入到选中节点的明细账目/成本中心/利润中心的网格中之后，将光标放到合适的地方 |
| 搜索跟随的节点 | 搜索跟随的节点的分组 |
| 搜索之前的节点 | 在之前的分组中搜索分组 |
| 展开节点 | 展开选中节点的细节 |
| 缩小节点 | 用于缩小选中节点的细节 |
| 增加选项卡 | 用于增加树的分组层次之间的间距 |
| 缩小选项卡 | 用于缩小树的分组层次之间的间距 |
| 搜索 | 用于搜索于结算的明细账目，为选中的节点可利用的 |
| 搜索明细账目 | 搜索于存在选中分组中的明细账目的列表 |
| 重新计算 | 用于重新计算重新分类的价值 |
