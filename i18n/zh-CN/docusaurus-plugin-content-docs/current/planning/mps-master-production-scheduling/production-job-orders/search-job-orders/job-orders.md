按照 计划 > 作业单 > 作业单的路径，用户可以打开作业单窗户。通过本窗户用户可以搜索已完成的生产作业单以及生成新的作业单（通过合适的按钮）。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 搜索 | 用于将搜索筛选器应用于 插入的生产作业单的作业单完全数据库 |
| 插入单一产品作业单 | 用于插入新的单一产品作业单 |
| 插入多个产品作业单 | 用于插入多个产品作业单 |
| 编辑作业单 | 用于编辑已插入的作业单 |
| 查看作业单 | 用于查看已插入的作业单 |
| 删除作业单 | 用于删除已插入的生产作业单 |
| 编辑作业单日期 | 用于编辑一个或多个生产作业单的结束日期  |
| 编辑属性的筛选器 | 用于打开 一个启动属性筛选器的窗户。通过这个筛选器用户可以按照属性筛选生产作业单。通过编码器用户可以插入作业单的属性 |

筛选器的区域：包含使用户做选择的数据类型清单。典型地，这些筛选器的状态是“AND”而且在同一时间内 用户可以确定多类标准的筛选器。

用户可以使用下列的筛选器：

**作业单类型** **.**:  通过本组合框用户可以插入生产作业单的类型以及按照这个标准筛选，因此查看选中类型的生产作业单；

**从编号 ….. 到编号**: 插入某个编号范围之后，通过这些字段用户可以搜索作业单；

**从年份 ….. 到年份**: 通过这些字段用户可以搜索作业单。用户按照作业单的年份，得插入时间的范围；

**项目**：在第一个组合框用户可以设置项目的类型。双击于第二个以及第三个字段之后，用户打开一个选择某个项目的“帮助项目”。按照本项目用户筛选生产的作业单； 

**变量**：在本组合框用户可以选择得筛选的项目变量；

**作业单类型**：通过本组合框，用户可以按单一产品、多个产品或者所有类型的作业单筛选；

**销售订单**：类型：通过本组合框用户可以设置订单类型、描述以及有关的编号。在这个情况下，用户可以查看与筛选器的订单编号有关的所有生产作业单；

**作业单**：通过某个合适的“帮助作业单”，双击于这个字段之后，用户可以选择与得搜索的生产作业单有关的销售作业单；

**客户**：在这些三个字段，用户可以插入客户的数据：“帐户”、“明细账目”、“客户的名称”。双击于第一个字段用户可以打开一个“帮助帐户”为选择客户。通过这种方式，用户可以搜索与选中客户有关的生产作业单；

生产地.: 通过这个字段，用户按照与作业单有关的生产地，可以筛选作业单；

**工作预测结束从日期....到日期：** 用户可以将工作预测结束的日期（从...到）插入到（在某个范围内）这些字段；通过合适的标记用户可以决定查看迟到/过期的作业单；

**采购类型**：通过这些标记，用户可以选择得筛选的采购类型，如果是采购作业单、生产作业单或者包工作业单；

**作业单状态**：通过合适的标记用户可以筛选要查看的作业单状态（未检查、计划的、启动、执行的、履行的、记录为历史）；

**结果** **网格**：结果网格表示相当于已确定的筛选器数据的生产作业单的清单。搜索参数确定了以后，通过掩码中的搜索按钮，用户能够获得需要的生产作业单清单；

结果网格表示的列子是：

**申请的类型**：如果是通过MPS定义自动地生成的作业单是依赖的，如果用户直接地生成地是独立的；

**作业单采购类型**：列子表示采购的类型：按照   在作业单MRP参数中设置的采购类型可以是生产、采购或者包工的类型；

**年份**:列子表示生产作业单的年份；

**工艺作业单：**列子表示与生产作业单有关的销售作业单的编号； 

**作业单的状态: **表示生产作业单的状态（未检查、计划的、启动、执行的、履行的、记录为历史）；

****工艺初始/结束的****预测****日期：****表示生产作业单的工艺初始/结束的预测日期；

**建议日期**:执行过程之前，如果在合适的参数中用户设置阻止日期，列子表示主计划建议作为结束生产作业单的日期； 

**分类**:表示属于生产作业单的项目分类；

**项目代码：**表示属于生产作业单的项目的代码 ；

**变量**:表示项目的变量；

**数量**: 表示项目的得生产的数量；

**客户：**表示与生产作业单有关的客户帐户/明细账目；

**作业单**: 表示与生产作业单行有关的销售作业单；

**项目说明：**表示得生产的项目说明；

**客户说明**: 表示于生产作业单行有关的客户合法名称；

**作业单类型的代码**:表示作业单类型的代码 ；

**作业单类型说明**: 表示生产作业单类型说明。
