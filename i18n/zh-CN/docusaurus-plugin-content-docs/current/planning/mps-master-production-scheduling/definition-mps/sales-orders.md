通过本表格，用户可以筛选被生产总计划表定义过程分析的销售订单行。本表格表示的订单行得符合下列的条件：

- 销售订单行的项目，在“物料需求计划”参数中得包括“作业单”标记激活；

- 通过领货清单、销售单或者销售发票用户不能履行订单行；

- 订单行的状态不能是“强制履行”；

- 订单行的状态得“未执行”或者“部分执行”；

- 订单行还没生成生产作业单；

- 订单行的销售订单的项目表格得包括现货日期；

- 包括行的销售订单状态得是“已打印l‘和”已确认“；

如果用户不符合上列介绍的所有条件，在网格中用户不能查看销售订单；

表格由**筛选器的区域和**结果网格组成。****

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 搜索 | 将搜索筛选器应用于完全的销售订单行的数据库。 |
| 按作业单考虑 | 用于将在”摘要“表格的网格中选中的销售订单行转移 |
| 流程生成 | 按照在”生产总计划表参数“窗户设置的参数，执行流程生成的过程 |
| 编辑属性筛选器 | 用于查看编辑属性筛选器。启动本按钮以及在弹出窗户的标记，用户可以利用属性的筛选器因此利用属于”编码器“表格的数据。 每个表格可以包括一个或多个筛选器。在结果网格中，用户可以查看与选中属性有关的列，以及属于表格的数据。如果用户利用属性筛选器，本网格表示属于属性窗户的数据。通过属于销售订单的项目表格的合适按钮用户可以启动本窗户。 |

筛选器的区域：包含使用户做选择的数据类型清单。典型地，这些筛选器的状态是“AND”而且在同一时间内 用户可以确定多类标准的筛选器。用户可以使用 按分类、按代码、按说明、按变量、按销售订单类型与编号、按客户、按销售作业单、按销售订单行的预测交货日期、以及按生产作业单的类型（在这个情况下，用户得筛选 从销售订单来的订单行。在”销售订单类型“，销售订单类型与某个作业单的类型有关）。

结果网格：结果网格表示相当于已确定的筛选器数据的销售订单行的清单。搜索参数确定了以后，通过掩码中的搜索按钮，用户能够获得需要的销售订单行清单。

结果网格表示的数据是：销售订单的类型、订单编号、订单年份、插入订单的日期（销售订单日期）、行号、表示帐户和明细账目的字段、销售作业单、分类、编码和项目变量、销售订单行的选项、还得履行的订单行数量（数量）、在订单行的现货日期（交货）中的可用数量（可用性）、代码、作业单类型的说明、项目说明以及客户的合法名称（客户的说明）。

 

如果属于”参数“表格的”考虑项目的可用性“标记不激活，”可用性“列的所有行等于0。 如果本标记激活的话，搜索订单行的过程需要更长的时间因为系统执行  每个项目行的  可用性的计算，按照设置的筛选器被可见的。

筛选需要的订单行之后，用户可以选择一些行（或者所有的），击于”按作业单考虑\“按钮之后可以将订单行转移到”摘要“表格。然后生成生产作业单的过程可以分析这些订单行。选择一个或多个销售订单行之后，用户可以利用”生成流“按钮。可是在生产总计划表的参数中，”自动生成生产流转“的标记得是激活。通过本按钮，生成生产作业单之后，用户可以直接地启动生产流程。这些作业单被计划以及按照标记，系统生成计划订单，生产订单以及生产的标记。
