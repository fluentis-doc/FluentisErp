在本表格中，用户可以设置以及保存发放计划订单的参数。对每个得发放的计划订单类型，表格由三个部分组成。

在**生产订单**的部分中，通过“为外部操作生成转包订单”标记，如果    计划订单的阶段列表包括外部阶段，系统对外部阶段生成包工的订单。普通地，本标记一直是激活。

在**采购申请**部分中，用户可以：

- 通过合适的组合框，设置采购申请类型。本申请被  采购的计划订单的发放部分生成。本组合框自动地表示物料需求参数包括的采购申请类型；

- 启动“采购申请上舍入” 的标记。通过本标记，发放采购计划订单的过程  将采购行包括的数量上舍入（如果有包括小数的数量）；

- 决定分组方法；通过“尽快分组”标记，发放的过程将    采购的计划订单分组。在    “日时期”  的字段中，用户可以设置   分组的   日范围。系统考虑预计接收的日期和采购的计划订单的结束日期；

- 如果采购的计划订单不包括供应商，用户可以接受未供应商的订单、忽略未供应商的订单或者将合适字段表示的供应商作为供应商。

在**转包商订单**的部分中用户可以：

- 通过 “订单类型”的组合框以及“年份”字段，设置转包订单的类型（关闭/开）和年份；

- 决定分组方法：通过“按同一转包商的订单分组”的标记，发放的过程将  计划的转包订单分组。过程将转包商得 管理 的项目插入到转包订单中；

- 如果转包的计划订单不包括转包商的话，忽略未转包商的订单或者将合适字段表示的转包商作为转包商；

本表格也包括使用户保存参数的图标。
