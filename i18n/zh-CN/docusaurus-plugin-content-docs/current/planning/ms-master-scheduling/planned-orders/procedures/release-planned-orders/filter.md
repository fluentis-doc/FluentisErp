在本表格中，用户可以查看得发布的计划订单的列表，通过一系列选择标注可筛选。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 搜索 | 用于将搜索筛选器应用于已插入的计划订单的完全数据库 |
| 发放生产计划订单 | 用于发放在结果网格中选中的生产计划订单。按照物料要求的参数，本过程生成启动的或者执行的生产订单 |
| 发放采购计划订单 | 用于发放采购的计划订单。用于发放在结果网格中选中的采购的计划订单。按照物料要求的参数，本过程生成授权/不授权的采购申请 |
| 发放转包商计划订单 | 用于发放在结果网格中选中的转包商计划订单。本过程生发放转包的订单 |
| 打开生产订单的搜索 | 通过在筛选器部分利用的标准，打开已筛选的生产订单的搜索。如果用户按照2012年的作业单编号3筛选计划订单，搜索生产订单的窗户为这个作业单已被筛选 |
| 属性 | 通过本窗户，用户可以启动属性的筛选器。本筛选器筛，按照编码器插入的属性，筛选计划订单 |

**筛选器的区域**：包含使用户做选择的数据类型清单。典型地，这些筛选器的状态是“AND”而且在同一时间内 用户可以确定多类标准的筛选器。

用户可以利用下列的筛选器：

**项目：**第一个组合框使用户设置项目的分类、双击于第二个字段用户打开某个“帮助项目”。通过本工具用户可以选择单一的项目（代码和说明）；

**变量：**双击之后用户可以打开某个“帮助变量”。通过本工具用户可以选择项目的变量；

**加工初始日期从..到：**用户可以将加工初始日期（在范围内）插入到这些字段； 

**加工结束日期从..到：**用户可以将加工结束日期（在范围内）插入到这些字段；

**维护订单：**通过本标记用户可以查看计划的维护订单；

**发放计划订单到：**在本组合框；用户选择“发放”之后生成的订单类型（采购申请、生产订单、包工订单）；

**订单类型：**本组合框包括计划订单的类型，按照本标准筛选，因此查看属于选中类型的计划订单；

**作业单类型：**本组合框包括生产作业单的类型，按照本标准筛选，因此查看属于选中类型的计划订单；

**生产作业单：** 这两个字段包括生产作业单的年份和编号；

**供应商：**这三个字段包括供应商的数据：帐户、明细账目、客户的合法名称。双击于第一个字段之后用户可以打开某个“帮助帐户”。通过本工具用户可以选择需要的供应商；

**作业单：**通过合适的“帮助作业单”，用户可以将销售作业单的编号插入到本字段；

**客户：**这三字段表示客户的数据：帐户、明细账目、客户名称。双击于第一个字段之后用户可以打开某个“帮助帐户”。通过本工具用户可以筛选筛选过程的客户；

**分组：**包括在网格中用户要查看的计划订单属于的分组编号；

**结果** **网格**：结果网格表示相当于已确定的筛选器数据的计划订单清单。搜索参数确定了以后，通过掩码中的搜索按钮，用户能够获得计划订单的清单。

结果网格表示的列是下列的：

**年份：** 表示计划订单的年份；

**订单：**表示计划订单的编号；

**层级：**表示识别计划订单的项目的物料清单的层级；

**订单类型的代码：**表示计划订单类型的代码；

**订单类型的说明：**表示计划订单类型的说明；

**作业单年份：**表示生成计划订单的生产作业单的年份；如果计划订单手动地被生成或者来自物料需求计划处理，本列是空的； 

**作业单编号：**表示生成计划订单的生产作业单编号。 如果计划订单手动地被生成或者来自物料需求计划处理，本列是空的；

**作业单类型的代码：**表示生产作业单类型的代码；

**作业单类型的说明：**表示生产作业单类型的说明；

**分类：**表示计划订单的项目分类；

**项目代码：**表示计划订单的项目代码；

**变量：**表示项目的变量；

**初始日期：**表示计划订单的预计初始日期；如果是采购或者包工的计划订单，本日期等于 用户将采购订单寄给供应商的日期。如果是生产的计划订单，本日期等于项目的加工路线的最后阶段的预计初始日期。

**结束日期：**表示计划订单的预计结束日期；如果是采购或者包工的计划订单，本日期表示  结束物料的日期因此等于采购订单行的的预计接收的日期。如果是生产的计划订单，本日期等于项目的加工路线的最后阶段的预计结束日期。

**数量：**表示计划订单项目的数量；

**确认的数量：**表示计划订单的项目的确认数量。如果计划订单来自物料需求计划处理的化，本数量与0不同；

**客户：**表示与计划订单有关的帐户和明细账户；

**供应商：**表示与计划订单有关的帐户和明细账目，可以是采购的或者包工的；

**特征：**表示计划订单项目的特征代码；

**作业单：**表示与计划订单有关的销售作业单；

**订单编号：**表示  生成生产作业单行的销售订单的编号。作业单的计划生成计划订单；

**销售订单的年份：**表示生成生产作业单行的销售订单年份。作业单的计划生成计划订单；

**销售订单类型：**表示生成 生产作业单行的 销售订单的类型。作业单的计划生成计划订单；

**项目说明：**表示计划订单项目的说明；

**客户说明：**表示与计划订单有关的客户合法名称；

**供应商说明：**表示与计划订单（采购的或者包工的）有关的供应商合法名称；

**特征说明：**表示计划订单醒目的说明；

**必须的：**包括一个标记。如果本标记激活的话，用户可以指定某个订单不能被转移。
