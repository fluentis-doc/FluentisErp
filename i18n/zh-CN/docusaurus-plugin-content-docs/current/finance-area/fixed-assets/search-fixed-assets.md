搜索固定资产的过程为容易地搜索已插入的固定资产而设计的。通过这个过程用户可以查看、编辑以及删除已插入的固定资产。在“搜索固定资产”中，用户可以完成关于某个文档的操作，相当的权利是必须的。

搜索固定资产的过程由下列的因素组成的。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 搜索固定资产 | 用于将搜索筛选器应用于已插入的固定资产数据库 |
| 新的固定资产 | 用于插入新的固定资产 |
| 编辑固定资产 | 用于编辑某个已插入的固定资产 |
| 查看固定资产 | 用于查看某个已插入的固定资产 |
| 删除固定资产 | 用于删除某个已插入的固定资产 |
| 拆分固定资产 | 仅当多个固定资产不选中，这个按钮激活。如果某个固定资产还每被折旧的话，这个按钮打开某个拆分固定资产的掩码。按照Ａ标头的应用 |
| 转移固定资产 | 从车辆模块创建某个机器 |

**筛选器的区域**：包含使用户做选择的数据类型清单。典型地，这些筛选器的状态是“AND”而且在同一时间内 用户可以确定多类标准的筛选器。** **

**结果** **网格**：结果网格表示相当于已确定的筛选器数据的固定资产的清单。搜索参数确定了以后，通过掩码中的搜索按钮，用户能够获得需要的固定资产清单。

标头应用：

A：拆分固定资产。在掩码中，用户得指定为每个公司部门／固定资产类别的得打开的表格数量: 为了确认按“是”，为取消按“不”。
