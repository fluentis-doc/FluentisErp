搜索会计记录的过程为容易地搜索已插入的记录而设计的。通过本过程用户可以查看，编辑，删除，插入新的会计记录。

通过 搜索会计记录的管理，用户可以完成关于记录的所有操作。相当的权利是必须的。

搜索记录的过程由下列特点组成的。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 搜索记录	 | 将搜索筛选器应用于已插入的会计记录完全数据库 |
| 新记录 | 用于插入新的会计记录 |
| 编辑会计记录 | 用于编辑已插入的会计记录 |
| 查看会计记录	 | 用于查看已插入的会计记录 |
| 删除发票 | 用于删除已插入的会计记录 |
| 原文档 | 用于利用生成会计记录的原文档 |

为了获得关于下列的报告数据，请按照相当的菜单项。



| RibbonFunction | RibbonMeaning |
| --- | --- |
|  [分析报告	](/docs/finance-area/ledger-records/analytic-reports/cost-centres-analysis)  | 用于打印关于下因素的情况：成本中心的分析、利润中心的分析、按成本中心的转移分析、成本中心/利润中心/作业单的失衡 |
|  [会计报告](/docs/finance-area/ledger-records/accounting-report/trial-balance)  | 按照筛选器标准，利用普通会计过程 打印：对帐单、试算表、注册、抵销、按模板的转移摘要、按文档类型的转移摘要、明细账目的会计情况、明细账目 |
|  [税报告	](/docs/finance-area/ledger-records/fiscal-report/period-balance-sheet)  | 按照筛选器标准打印：财务报表、 日记账，增值税登记 |
| 财政报告	 | 按照筛选器标准打印；会计余额和daoriqi 的差异、收款/支付的平均延期、partite aperte的 对帐单、partite列表、 partite的分组 -到日期的摘要 -	客户/供应商的票据账簿S |

为了获得关于下列的过程数据，请按照相当的菜单项。



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 结算交易 | 用于管理属于多样明细账目的所有交易结算  |
| 交易分组 | 用于管理一个或多个交易的新组 |
| 结算账户	 | 用于管理交易的收款  |
| 调整分录	 | 用于管理年份底的调整分录 |

**筛选器的区域**：包含使用户做选择的数据类型清单。典型地，这些筛选器的状态是“AND”而且在同一时间内 用户可以确定多类标准的筛选器。** **

**结果** **网格**：结果网格表示相当于已确定的筛选器数据的记录清单。搜索参数确定了以后，通过掩码中的搜索按钮，用户能够获得需要的记录清单。
