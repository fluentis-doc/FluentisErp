查看账户的掩码使用户查看任何账户/明细账目的会计分录，包括任何的类型（贷方、债务、成本、利润或者登记）。会计年度的分录一直对齐。按照期限结算，这些账户可以算必须的税报告或者结算账户的操作。

在上面的部分中用户得插入一个账户代码或者详细的明细账目：下列的筛选器字段是可选的。它们涉及预定的筛选器和最终记录的标记。

网格默认地表示按照设定筛选器的分录列表（按日期）：如果系统按记录日期 年/01/01  筛选，“以前余额”部分的上面字段表示比 年/01/01日期前的记录贷方/债务总额。 在记录网格中，“顺序”字段的第一个行    表示  本总额，按行价值 更新。然后本字段是一行一行被更新的（注意：这个顺序字段，  仅当按记录日期或者应记日期的升序有意思。在其他的情况下没有意思）。掩码基础的总额表示：“选择余额”等于在网格中选中的贷方/债务的总计，“当前余额”等于在网格中选中的贷方/债务的总计，“总额”等于上个余额和当前余额的贷方/债务的总计，“余额”包括贷方/债务的最终余额（相当于最后行表示的顺序）。

在“查看账户”层级中可以完成的操作是：管理网格的时候，得考虑到的因素是两个表示的明细账目：标头明细账目（来自记录的持有人的明细账目）详细明细账目（系统搜索详细明细账目的分录）。如果用户在查看普通的成本账户的分录，标头明细账目表示送给我们的已记录的发票 的供应商代码。详细的明细账目表示成本的明细账目。

- 双击于单一的行：在这个情况下，如果有相当的权利，系统打开编辑分录属于的记录。

- 通过“编辑”按钮用户可以获得 刚介绍的结果。

- 通过“原文档”按钮，（仅档有相当的权利）用户可以查看   与会计记录一起生成选中的会计分录的销售/采购发票

- 通过“详细”按钮（仅当一个行激活）用户在一个分开的网格中可以查看会计记录的所有分录。 用户将光标转移到更高的记录，网格不工作。

查看账户的过程由下列的因素组成的。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 查看账户 | 用于查看已插入的会计记录 |
| 编辑 | 用于编辑已插入的新会计记录 |
| 原文档 | 用于利用生成会计记录的原文档 |
| 详细 | 在结果网格中表示选中的会计记录详细 |

 

**筛选器的区域**：包含使用户做选择的数据类型清单。典型地，这些筛选器的状态是“AND”而且在同一时间内 用户可以确定多类标准的筛选器。

**结果** **网格**：结果网格表示相当于已确定的筛选器数据的记录清单。搜索参数确定了以后，通过掩码中的搜索按钮，用户能够获得需要的记录清单。
