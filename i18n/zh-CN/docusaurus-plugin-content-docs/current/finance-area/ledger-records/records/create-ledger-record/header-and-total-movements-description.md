记录标头的字段是下列的（按装载顺序）：

**记录日期**：在新的记录中，本日期以系统的当前日期被设定。除了增值税凭证，已打印的时期和参数之外，用户可以改变本日期。本日期是非常重要的，因为用户不需要“会计年份的改变”。从记录日期以来，应用搜索参考会计参数的年份（识别着会计年份的日期范围）。通过本年份，用户可以利用其他监控日期的范围（当前应记日期和当前时期）。本日期是账簿和会计对帐单的根据日期。如果“新记录”按钮激活的话，用户可以设定用户参数。通过这个方式，系统表示在新的记录中表示这个字段。

**记录编号**：是记录的每日计数器。记录的唯一性来自“记录日期+记录编号”。每日计数器给用户记录于多日期的可能。 过程自动地管理本记录编号。 保存的时候系统设定最终的正确编号。系统解决编号的缺乏的问题。删除某个日期的记录导致关于这个日期的编号缺乏。插入/编辑某个记录之后，系统可以解决编号缺乏的问题 。在临时的记录的情况下，该编号的级数从-1000开始（负数）。通过这种方式用户可以清楚地查看本状态的记录。报告和会计记录默认地筛选最终记录。

**应记日期**：是会计年份的参考日期。本日期通常等于记录日期（不包括财务报表的调整分录以及有关的账户打开/关闭操作。这些操作的日期记录可以是六月可是应记日期为12月31日-1月1日）。这个日期对调查得接收的发票没有用。最好是这个日期等于记录日期。在会计中，一些报告按记录日期，其他按照应记日期，其他按照两个日期类型。按照利用的筛选器用户获得不同的结果。在管理用户的参数，用户可以设定应记日期按记录日期自动地对齐。

**记录的明细账目:**该字段对增值税记录是必须的，因为它与已记录的客户/供应商登记有关。在支付/收款记录的情况下，用户通常不需要填写这个字段。通常，在同一的记录中，用户完成关于不同客户的一些支付/收款，在“支付”中这个字段以交易的筛选器为利用的。如果用户按“生成新记录”，在管理的记录中可以设定用户参数。通过这中方式在新记录中系统表示这个字段。 会计模板涉及 已插入的账户类型与模板包括的一致性检查（通知或者阻止）。按照会计参数，客户/供应商部分，系统授权替代。

 模板：本字段是必须的。每个记录按照某个参考的模板。通过这个模板用户可以决定每个记录的特点和功能。保存某个记录之后，用户不再编辑这个字段 (明显地，如果增值税登记/者账簿已打印或者账户的结算完成了，系统阻止其他字段如果模板是错的，用户得删除然后再插入包括正确编码的记录。如果用户按“生成新记录”，可以再设定用户参数。通过这个方式，在新记录中系统表示这个字段。 

**增值税登记**：选中的会计模板表示字段。用户可以直接地编辑这个字段可是这个操作不建议的 （更好是编码属于第二个增值税登记的新的模板）。

**增值税凭证**：数字字段，被过程自动地管理。打开掩码的时候，系统表示登记的+ 1最大凭证。保存的时候，系统按照某个增值税登记/凭证的别的可能保存，确认这个凭证。关于凭证检索：如果模板的不连续编号阻止保存已设定的话检索类型改变。如果“阻止”激活的话系统表示某个有同一记录日期的自由凭证，要不然系统表示在那个日期无效的凭证，包括有关的通知）。在临时记录的情况下，凭证的字段不工作。

**文档类型**：按照普通会计模板的价值，过程表示这个字段。是可编辑的字段可是通常用户不需要编辑这个字段（典型地，用户得指定增值税记录的文档可是这个数据已经在模板）

**文档日期**：按照会计模板的设定，这个字段可以是必须的（属于第二个表格的文档日期标记）。不仅这个日期不能超过记录日期而且在关于记录会计年份的当前时期 的有效日期的范围内。如果用户按“生成新记录”，可以设定用户参数。通过这种方式，在新记录中系统表示这个字段。

**文档编号**：是字母数字的20字的字段。按照模板的设定，为记录标头明细账目内的编码，系统可以阻止同一编号、同一年份（按文档日期）、同一类型（文档类型字段） 的插入操作。这种阻止在每个会计就模板默认地被设定，但是用户可以取消它。如果编号/年份/明细账目同一可是类型不同，系统简单地通知用户。同样地，如果参考在临时记录保存的话，系统通知用户。

**作业单字段**：如果会计模板涉及作业单的管理，这个字段激活。这个是完全记录的参考作业单。在“作业单”内用户可以再指定价值。为了填写字段用户得利用“帮助作业单”。通过这个方式用户可以筛选属于销售区域的缉获销售作业单。

**货币**： 被记录的登记表示的 ，作为公司货币的。用户可以每次编辑这个字段。保存记录之后，用户不再编辑这个字段。

**货币日期**：是汇率的参考日期，按照汇率表格。过程按   比这个日期前的最后日期。在管理用户的参数中，这个日期可以是记录日期（默认）或者文档日期。记录日期的时候，如果汇率不设定的话，系统按照一个用户的通知参数。

**汇率**：表示货币汇率表格的价值。本字段表示与普通货币汇率表格比较相反的价值（因为过程还按照以前的里拉逻辑。欧元按照其他逻辑：1欧元=多少其他货币单元）。用户可以直接地强制这个价值（典型的情况：支付/收款的货币模板。银行利用的汇率不按照国际汇率的）。

**货币总额/文档总额**：汇率联系这两个字段因此在欧元记录中用户可以随便插入于货币总额/文档总额。根据会计模板的设定，按增值税网格的数据改变，这两个字段可以自动地更新（增长/降低）。如果这个总额与记录的增值税的数据不同，系统阻止记录的保存。

**税总额**：过程自动地更新这个字段

在会计记录的下部分中，在记录部分中，有一个会计记录的明说。 在会计模板中，通过1) (2) … (10) 的代码，用户可以设定这个票据（ (8) 和 (9)代码关于属于账簿部分的明细账目的详细明说）双击于字段，用户可以开始搜索于编码的票据。 "工具","效用"包括这个过程): 票据管理的字段编辑之后，这个字段的直接编辑按用户的确认复位。
