仅当模板的“打开交易”标记激活，交易表格存在。

交易表格的第一个部分管理下交易的计算的数据。增值税类型的记录网格包括插入到客户/供应商/代理商登记中的默认数据。关于别的记录类型：例如：为预支付，用户可以编码某个打开交易的模板，利用相反的符号，而非关闭它因为文档还没被调查): 在这样子的情况下，这个部分的管理成为动手的。用户可以直接地插入付款，付款方式以及参考的明细账目的主要数据。“百分比”字段包括100%。

关于这个部分的值得注意因素是：

- 用户可以将某个交易生成于不属于登记的明细账目。这些明细账目的会计科目表不包括“打开交易”  标记（这个标记的责任是监控账簿和交易的数据与有关的明细账目的数据相当的）。

- 典型地，登记或者被   采购/销售区域     已会计的文档表示参考的银行。通过帮助IBAN用户可以强制参考的银行。与这个部分有关的参考银行是     供应商支付通常地利用    的数据，作为收款的银行，除非“利用属于供应商登记的参考银行”标记激活的话。

- 通常地，“百分比”的字段被插入的。金额/税的意思是：“金额”字段可以包括对付款类型/付款方式的组合利用的固定的金额。“税”字段指定与增值税有关的金额（如果付款方式涉及 分期付款的   应纳税/税百分比，这个功能很有用）。用户可以插入 金额/百分比 的组合（已在登记中）。通过付款类型/方式，过程限价交易，覆盖着金额。关于剩余，按替代行的百分比计算（例如：1000金额/30日银行转帐,100百分比60/90银行收据。如果金额等于1500，系统生成 1000金额/30日银行转帐   的某个交易,  250金额和0/90银行收据的两个交易）。

- 在左边的“分组交易”标记被客户/供应商登记利用的：没有实际使用因为在某个文档中，两个交易具有同一付款方式以及同一到日期的情况很少发生的。

- 下部分包括一些摘要客户/供应商的情况：在信用风险模块中的透支金额，按照打开的交易情况获得的   价值（不考虑到在使用中的记录）， 会计余额（考虑到在使用中的记录）。

- 从未付会计来自的记录：在这个情况下，上面的网格不影响下面交易的计算（典型地，登记涉及  银行收据的交易，可是未付作为一个银行转帐管理的，因此银行收据的 自动发布不是必须的）。如果客户需要参考银行的数据，它很有用。

下面网格表示按上面的设定计算的交易列表。 用户可以强制付款类型，到日期（为了将它相当于文档的数据）和分期付款的金额（同时考虑到总额）。更有意思的是直接地阻止交易付款的可能：过程记录日期以及阻止付款的用户（包括解释的票据 ）以及解锁付款的日期/用户（这两个操作需要属于ArtinfoUserManager的特别用户权利）。别的手动的管理因素是：交易位置（用户可制定的自由表格）。通过这个方式用户可以分类会计交易，从生成的时候。交易的票据：在会计模板中它们自动与明细账目的详细明说自动地联系。 交易网格不允许用户插入新的分期付款。 通过最后的字段用户可以分配（或者编辑）交易的参考代理商。按照会计记录的时候插入到销售发票的代理商，这个字段自动地限价（如果有很多的代理商，系统指定更高佣金的代理商）。如果模板涉及交易的打开以及 结算的关闭 而且“通知”标记激活，在保存的时候系统通知用户。
