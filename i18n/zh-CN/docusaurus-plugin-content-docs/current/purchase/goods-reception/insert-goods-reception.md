title: 货物接收的管理
sidebar_position: 3
---

为了插入新的货物接收，用户得插入：

**供应商**：通过帮助或者账户/明细账目的直接插入

**日期：**表示等于当前日期的日期

**编号**：表示日期最后的有用序号

**类型**：货物接收类型

可插入到货物接收的项目类型只是在登记中编码的项目。用户可以直接地插入项目或者将它们从条形码/采购订单导入到系统。

**代码**：指定在项目中的“项目条形码”，“条形码”表格。插入的时候，系统自动地检索有关的项目以及登记的所有数据。用户可以直接地或者通过“帮助项目”（双击于黄色部分）插入分类或者编号。黄色部分表示项目登记的有关数据。插入项目之后，登记自动地检索项目的明说。如果项目有变体的话，在“变体”组合框中用户得选择需要的变体；

**变量**：表示选中项目的变量。选择变体对具有这个特别配置的项目很有用。这些项目的价格与标准价格可不同。因此，有变体的项目价格与未变量的可不同。这个可需要在参考的销售价目表中的项目变量管理；

**供应商的项目**：按照供应商项目的表格。用户可以直接地或者通过帮助插入项目；

**供应商的项目变体**：表示供应商的项目变体的表格。用户可以直接地或者通过帮助插入项目；

**计量单位**：表示项目的主要计量单位可是用户可以选择替代的计量单位；

**文档数量**：表示主要计量单位的数量。插入已履行的采购订单的数量，不是必须的数据；

**收到数量**：表示收到数量；

**替代的计量单位**：表示属于项目登记的替代计量单位表格中的计量单位。“默认”标记是激活的；

**替代数量**：表示替代计量单位的价值。系统将管理的数量乘／除以在项目登记中设置的缩放比例；

**价格**：表示在采购订单中的价格或者项目登记的最终成本；

**选项**：必须的数据，相当于物料清单选项的表格；

**仓库**：是接收货物的仓库。如果接收行履行采购订单行，采购订单检索接收的仓库。用户也可以直接地插入接收的仓库；

**模板**：货物接收的模板。如果接收行履行采购订单行，采购订单检索货物接收的模板。用户也可以直接地插入货物接收的模板；

**项目明说**：表示项目的明说。用户不能编辑项目明说；

**订单的参考**：如果　　履行采购订单行的程序生成行，系统表示已履行的订单编号；

**库位**：如果模板有默认的库位，系统表示本库位。如果仓库按照库位被管理，用户可以直接地插入库位；

**作业单**：用户可以将每个接收行与销售作业单联系。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 保存货物接收 | 保存接收 |
| 新的项目 | 为了执行新的插入用户将它放在项目网格中， |
| 删除项目 | 删除选中的项目行 |
| 项目属性 | 如果项目行通过本功能履行某个采购订单行，用户可以查看参考采购订单的数据以及属于摘要项目行的数据 |
| 从采购订单 | 通过采购订单的履行，过程生成接收行 |
| 从条码导入 | 从一个在参数中配置的表格，导入项目行 |
