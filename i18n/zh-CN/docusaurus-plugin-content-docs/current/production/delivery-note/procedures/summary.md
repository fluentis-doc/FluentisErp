在本部分中，用户可以恢复送货单的记录操作，因此可以恢复操作之前的情况。本表格的标头包括一些搜索筛选器。通过这些筛选器，按照缺载编号（从...到）、缺载送货单的日期（从...到）和送货单编号（从...到），用户可以选择操作。

属于网格的“缺载送货单”窗户的上部分表示记录操作的编号、操作的日期以及用户。如果用户选择本网格的一个行，掩码的下部分可以表示已缺在的交货单（在合适的“送货单”表格，包含年份、送货单类型、送货单类型的说明、编号、送货单日期）以及有关的仓库记录（在合适的“记录”表格，包括仓库、仓库模板、仓库记录的编号和日期、转包商、参考的文档、文档日期以及抵销记录的数据）。

属于功能导航栏的按钮对恢复的操作很有用。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 搜索 | 将搜索筛选器应用于 已插入、已打印以及已缺载的送货单的完全数据库 |
| 删除 | 用户恢复   在“缺载送货单”网格选中的送货单的完全缺载操作。因此，如果同一的操作缺载多送货单、这些货单的状态是“未缺载” |
| 恢复送货单 | 用于恢复在下网格的“送货单”表格中选中的送货单的记录 |
| 恢复 | 用于删除  在“记录”表格选中的仓库记录，不表示有关的包含“未缺载”状态的送货单。除非特别的情况发生，我们建议普通地不利用本按钮 |
| 报告 | 预览缺载在“缺载送货单”网格选中的送货单的操作。表示已缺载的送货单的详细以及系统还没缺载一些送货单的原因 |
