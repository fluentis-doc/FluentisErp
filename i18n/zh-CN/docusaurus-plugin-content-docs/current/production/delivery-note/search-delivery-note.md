title: 搜索送货单
sidebar_position: 2
---

为了打开送货单的窗户，请按照转包 > 送货单 > 送货单 > 送货单的路径。本窗户表示    已插入的转包送货单列表。用户可以查看、编辑以及插入新的送货单。 

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 搜索 | 将搜索筛选器应用于已插入的转包送货单的完全数据库 |
| 插入 | 用于插入新的送货单 |
| 编辑 | 用于编辑某个送货单 |
| 查看 | 用于查看送货单 |
| 删除 | 用于删除已插入的送货单 |
| 编辑属性筛选器 | 打开一个窗户。通过本窗户用户可以启动属性筛选器。本工具按照通过编码器插入的属性，筛选转包的送货单 |

窗户由筛选器和结果区域组成。

筛选器的区域：包含使用户做选择的数据类型清单。典型地，这些筛选器的状态是“AND”而且在同一时间内 用户可以确定多类标准的筛选器。

用户可以利用下列的筛选器；

**转包商：**这两个字段表示送货单的转包商的数据：“帐户”以及“明细账目”。转包商的合法名称自动地出现。双击于第一个字段之后，用户可以打开某个“帮助帐户”然后选择转包商。普通地，转包商属于会计科目表（其中供应商） 

**退回类型：**本组合框可以包括货单的类型然后按照本标准筛选，因此查看属于选中类型的所有送货单。组合框表示 销售单的所有类型 的列表（因为从管理的角度，送货单 与销售送货单可互相合成）；

**从货单编号… 到货单编号…：**包括货单编号，从...到（在某个范围内）；

**从货单日期… 到货单日期…：** 包括插入送货单日期，从...到（在某个范围内）；

**作业单：**通过合适的“帮助作业单”，用户可以将销售作业单的编号插入到本字段；

**已卸载：**通过本标记用户可以筛选从仓库已缺载的送货单。对这些货单缺载货单的操作已完成；

**已打印：**本标记使用户筛选已打印的送货单。为了打开搜索货单的窗户，最后两个的标记不激活。为了查看所有的货单用户得启动这些标记；

**搜索项目：**通过本按钮，用户可以启动    使用户筛选的部分。筛选的过程按照下列的参数；

**分类/项目代码/说明：**第一个组合框使用户设置项目的分类，双击于第二个字段之后用户可以打开某个“帮助项目”。通过本工具用户可以选择单一的项目（代码和说明）；

**作业单/年份：**通过合适的“帮助作业单”，用户可以将销售作业单的编号插入到本字段。在这个情况下，就是与送货单的项目的单一行有关的销售作业单；

**作业单类型：**用户可以将生产的作业单类型插入到本组合框。用户也可以按照本标准筛选，因此查看 与选中类型的作业单  有关   的所有退回；

**生产作业单：**这两个字段包含生产作业单的编号和年份；

结果网格：结果网格表示相当于已确定的筛选器数据的送货单的清单。搜索参数确定了以后，通过掩码中的搜索按钮，用户能够获得需要的送货单清单。

结果网格表示的列是下列的：

**已打印：**通过本标记用户可以识别如果某个货单已打印。如果货单未打印，系统不能完成缺载的操作；

**已缺载：**通过本标记，用户可以识别如果货单已缺载；

**编号：**表示送货单的编号；

**日期：**表示送货单的日期；

**转包商：**表示送货单的转包商的帐户和明细账目； 

**转包商说明：**表示送货单的转包商的合法名称；

**总计：**表示交货总计，按照增值税率。
