通过本表格，用户可以执行操作的“恢复”因此恢复以前的情况。本表格的标头表示一些搜索筛选器。通过这些筛选器，按照缺载编号（从/到）、退回记录日期（从/到）、退回单编号、转包订单编号（从/到）、退回的转包商，用户可以选择操作。

窗户的上部分，叫“缺载退回”，表示过程的结果。本部分也表示记录操作的编号、日期和用户。如果用户选择本网格的某个行，在掩码的下部分中用户可以查看记录的退回（在合适的退回表格，表示转包年份、编号以及日期）以及有关的仓库记录，被退回记录自动地生成（在合适的“记录”表格，表示仓库、仓库模板、仓库记录的编号和日期、转包商、参考文档、文档日期、抵销记录的数据）。 

通过属于功能导航栏的按钮用户执行恢复的操作，按照不同的方式。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 搜索 | 将搜索筛选器应用于插入的、受控以及已记录的转包退回的完全数据库 |
| 删除 | 用于恢复在“”网格选中的记录退回的完全操作。因此，同一的记录操作记录很多退回。所有的退回状态是“未装载” |
| 恢复送货单 | 用于恢复在下面网格选中的退回记录 |
| 恢复 | 用于删除在“记录”表格选中的仓库记录，不表示有关的包含“未装载”状态的退回。除非特别的情况，我们建议用户不使用本按钮 |
| 报告 | 用于预览在“缺载退回”网格选中的记录退回操作。表示记录退回的详细以及它们还没记录的原因 |
