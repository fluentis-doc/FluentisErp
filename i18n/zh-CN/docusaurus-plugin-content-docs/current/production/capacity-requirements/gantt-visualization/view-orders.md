---
title: 查看订单
sidebar_position: 4
---

“查看订单”窗户由两个相同的表格组成。每个表格由两个网格组成。较低网格按照用户在较高网格中选择的因素。

第一个叫“在甘特图的订单”表格的较高网格表示 每个生产订单的每个阶段的一个行。

结果网格表示的列：

**订单类型：**表示文档的类型；可以是”执行的“、”启动的“或者”计划的“；

**生产作业单：**表示生成订单的生产作业单的年份和编号。如果生产的订单手动地被生成或者来自物料需求计划处理，本字段是空的。

**作业单类型的代码：**表示作业单类型的代码；

**作业单类型的说明：**表示生产作业单类型的说明；

**订单：**表示计划/生产订单的编号；

**订单类型的代码：**表示计划订单类型的代码；

**订单类型的说明：**表示计划订单类型的说明；

**订单初始日期：** 表示计划/生产订单的预计初始日期。本日期是项目路线的第一个阶段的预计初始的日期； 

**订单结束的日期：**表示计划/生产订单的预计结束日期；本日期是项目加工路线的最后阶段的预计结束日期；

**执行订单：**如果订单还没有生产报告的话，列的价值等于0。如果订单已经有生产的报告价值等于1；

**分类：**表示订单项目的分类；

**项目代码：**表示订单项目的代码；

**项目说明：**表示计划订单项目的说明；

**变量：**表示项目的变量；

**变量说明：**表示项目变量的说明；

**代码 / 交货区：**表示交货区的代码和说明；

**生产阶段：**表示生产阶段的阶段以及子阶段的代码;

**阶段说明：**表示生产阶段的说明；

**工作中心的时间：**表示工作中心应用于那个阶段的完全时间

**%：**表示时间的百分比价值，关于某个天可用的生产能力

**项目、说明、变量、说明、第一个层级的变量 ：**表示关于属于物料清单的第一个层级的粗制品的数据；

**数量：**表示项目的计划订单的数量；

**确认的数量：**表示项目的计划订单的确认数量。如果计划订单来自物料需求计划的处理，本数量于0不同；

**客户：**表示与计划订单有关的客户帐户和明细账目；

**供应商：**表示与计划的采购/转包订单有关的供应商帐户和明细账目；

下列的网格表示生成生产作业单的销售订单的行。这些作业单的计划生成计划/生产订单，在较高网格中出现。 系统表示下列的数据：类型、年份、编号、插入销售订单的日期、属于销售订单标头的客户、分类、代码、说明、属于销售订单行的项目变量和变量说明、预计交货的日期。

第二个表格表示“不在甘特图”的订单，结构等于“在甘特图”表格的。

通过一个小的功能导航栏，用户可以替代选中文档的工作中心以及编辑在较高网格中选中的文档日期。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 导入模拟 | 用于确认关于 图形文当的用户完成的编辑 |
| 保存模拟 | 将用户执行的模拟保存于某个文件，可是图形的文档不包含这些编辑 |
| 装载模拟 | 将已保存的模拟转载于图形 |
| 缩小 | 用于提高图形的时间详细 |
| 放大 | 用于减少图形的时间详细 |
| 物料需求计划 | 使物料需求计划处理于完全的数据库 |
| 过载的工作中心 | 打开某个包括一个网格的弹出窗户。本网格包括  图形文档 的工作中心。工作中心超过自己的生产能力的天是红色  |
| 恢复模拟 | 用于恢复模拟 |
