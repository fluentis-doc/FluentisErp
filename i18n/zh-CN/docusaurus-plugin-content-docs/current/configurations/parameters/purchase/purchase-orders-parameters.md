title: 供应商订单的参数
sidebar_position: 5

通过供应商订单的参数用户可以正确地管理，按照每个公司的申请。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 恢复参数 | 用于恢复初始的参数 |

#1.1 总表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 双重计量单位管理 | 激活的时候，在订单项目的网格，替代计量单位和替代数量列开始工作 |
| 自动建议替代计量单位 | 仅当“双重计量单位管理”激活的话，本参数开始工作。如果用户将某个登记包含替代计量单位表格以及某个默认计量单位的项目插入，本计量单位作为项目的自动计量单位。通过 缩放比例，系统可以计算替代数量 |
| 仅在某一天中恢复 | 激活的话使用户在某一天中恢复编号的空。如果在某一天中7,8,11,12订单被生成，下的建议编号是10，自由的编号，比最后的编号小 |
| 在变量属性中自动地插入变量 | 激活的时候，在管理订单的项目表格中，变量代码和变量说明成为两个可编辑的字段。系统问用户如果想将数据保存于变量登记中。按这个方式，插入订单行的时候，系统省城项目的变量  |
| 不允许执行更大数量 | 如果激活的话，生成某个“从订单接收物料”的时候， 用户不能插入比本订单大的数量 |
| 监控日期和数字次序 | 如果激活的话，文档日期和数字次序是保证的。因此，如果2011/11/21日期包含7, 8, 10订单以及2011/11/22 日期包括11号订单，为了保证日期/次序次序用户只可以插入12号订单  |
| 无价目表成本为0 | 激活的时候，如果订单项目的价目表没有，成本为0。如果不激活的话系统建议项目登记的最后成本 |
| 查看规模 | 激活的时候，管理采购单的表格表示三个列：高度、宽度、深度（MGAD_高度、 MGAS_宽度、MGAS_深度，从MG_数据分析 – 项目登记的中的重量/规模表格） |
| 阻止日期为公众假日的文档插入 | 如果激活的话，用户不能插入包含假日的文档 |
| 仅履行授权的订单 | 如果激活的话，履行订单的时候，用户只可以查看授权的订单 |
| 建议有采购/销售订单价目表优先级的品牌 | 如果激活的话，系统将订单插入到订单，系统保存订单价目表项目的品牌。如果本参数激活的话，从销售订单的供应商生成，将订单品牌复制于采购订单 |
| 允许编辑增值税 | 如果激活的话，在“从采购申请订单的自动生成” 中，增值税代码在订单中被建议 |
| 操作员代码为必须 | 如果激活的话，用户得插入操作员/员工 |
| 在所有预定义价目表中搜索项目价格" | 如果激活的话使用户搜索供应商价目表的价格。搜索关于在供应商登记预定义的类型的价目表，无论价目表无效（有效的价目表的），有效价目表的有效初始日期<= 当前日期，有效结束日期为0或者>= 当前日期 |

#1.2 从条形码导出的表格在本表格中用户可以设置   为了导出属于 OF_ImpBCAll  表格的数据的参数。为了打开导出表格，用户得打开  项目表格的物料接收管理的工具兰。

#1.3 装载表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 创建有文档日期的记录 | 如果激活的话，仓库记录的日期等于接收物料的日期。在装载表格中用户不能指定记录日期 |
| 接收参数优先级 | 如果本参数激活的话，仓库和装载模板被物料接收行读（如果没有装载表格的参数），如果不激活的话系统利用插入到本参数表格的仓库和模板 |

#1.4 分析表格这些参数指定   成本/利润中心的搜索优先，然后将这些中心分配于供应商订单的项目。如果重新计算标记激活，通过选中的优先，成本/利润中心在项目中被插入/更新。
