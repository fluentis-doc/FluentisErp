---
title: 销售发票的参数
sidebar_position: 5
---

销售发票的参数使基础设置正确地管理，按照每个公司不同的需要。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 恢复参数 | 用于恢复初始参数 |

#1.1 总表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 建议所有项目文档中插入的佣金 | 系统不再管理本参数 |
| 插入佣金到行类型6和7（预付款/冲销) | 预付款/冲销的项目行管理代理商 |
| 双重计量单位管理 | 如果激活的话，替代计量单位组合框以及在发票管理的项目的网格中的替代数量列 开始工作 |
| 自动建议替代计量单位– | 仅档“双重计量单位管理”参数激活的话，本参数开始工作。用户插入某个包括默认计量单位的项目的时候，本计量单位作为项目的替代计量单位。通过计量单位的缩放比例系统也计算替代数量 |
| 自动建议承运人负责的运输 | 如果激活的话，运输类型是承运，系统建议默认的承运 |
| 容量的默认计量单位 | 如果激活的话，在货单的运输计算中，作为容量的默认计量单位 |
| 重量的默认计量单位 | 如果激活的话，在货单的运输计算中，作为重量的默认计量单位 |
| 仅在某一天中恢复 | 如果激活的话，只在当前日期中，系统恢复可用的编号 |
| 自动恢复 | 如果激活的话，系统自动的表示第一可用的编号 |
| 监控可用性 | 如果激活的话，系统监控货单项目的可用性 |
| 借记单上的负价格 | 如果激活的话，在包含借记单类型的发票中系统表示正价格 |
| 排除销售订单 | 如果激活的话，货单项目可用性的计算不包含销售订单的使用 |
| 必需的可用性 | 如果激活的话，货单项目的可用性是必须的，因此数量不能大于可用性 |
| 查看可用性 | 如果激活的话，用户将数量插入到行的时候，系统表示项目的可用性 |
| 转包 | 如果激活的话，货单项目可用性的计算包含转包文档 |
| 采购 | 如果激活的话，货单项目可用性的计算包含采购文档 |
| 销售 | 如果激活的话，货单项目可用性的计算包含销售文档 |
| 计划的生产 | 如果激活的话，货单项目可用性的计算包含物料需求文档和生产作业单 |
| 已发布的生产 | 如果激活的话，货单项目的可用性计算涉及生产订单 |
| 仓库 | 如果激活的话，货单项目的可用性计算涉及仓库记录 |
| 检查下一个价目表 | 系统将插入到发票行的项目搜索于客户登记价目表和默认的价目表 |
| 监控在欧洲经济共同体中 | 监控如果发票类型和客户有同一的“在欧洲经济共同体中”类型 |
| 条形码使用 | 系统管理属于项目网格的项目条形码列 |
| 尺码和颜色管理 | 如果激活的话，系统管理变量网格的批次列 |
| 查看规模 | 如果激活的话，系统管理项目网格的规模列（高度、宽度、深度） |
| 允许礼品费的折扣 | 如果激活的话，礼品管理折扣 |
| 监控完整性 | 如果激活的话，监控发票的完整性 |
| 重算运输 | 系统自动地计算发票的运输 |
| 无手续费的收款单 | 包含借记单类型的发票到日期不包含收款费  |
| 阻止日期为公众假日的文档插入 | 如果激活的话，系统阻止假日期的使用 |
| 核实销售价格批次 | 系统监控发票行的单总额 |
| 用户代码为必须 | 如果激活的话，发票标头中的用户代码是必须的 |
| 订单外部参考 | 系统管理与发票项目行有关的销售订单的外部参考 |
| 最大行数量 | 指定项目网格中的最大数量 |

#1.2 履行表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 支付 | 在订单的多履行的情况下，表示 发票支付的利用价值：第一订单、登记或者手动选择 |
| 收件 | 在订单的多履行的情况下，表示发票收件的价值：第一订单、登记或者手动选择 |

#1.3 缺载表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 创建有文档日期的记录 | 如果激活的话，仓库记录包含送货单的日期，然后在缺载表格中，用户不需要指定记录路日期 |
| 优先级仓库和装载模板  | 如果本参数激活的话，发票行读仓库和缺载模板，如果不激活的话，系统利用已插入到本表格的仓库和模板 |
| 仓库 | 仓库首先和从送货单项目行缺载模板不激活的时候，表示发票缺载得利用的仓库 |
| 模板 | 仓库首先参数和从送货单项目行缺载模板不激活的时候，表示发票缺载得利用的模板 |
| 通知虚拟项目 | 如果激活的话，缺载发票的时候，系统监控如果发票包含虚拟项目。如果有的话，某个问用户缺载发票的信息出现 |

#1.4 导入表格在本表格中，为了将项目从条形码导入，用户指定文件成名和路线。为了打开导入表格，按照发票管理的工具兰，在项目管理中。通过本导入操作用户可以将文件的项目插入到发票。

#1.5 条形码导入的表格在本表格中，用户可以指定 为了导入属于FT_ImpBCAll表格的数据的参数。为了打开导入的表格，用户按照发票管理的工具兰，在项目表格中。通过本导入的操作，用户可以将项目插入到发票。



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 替代计量单位 | 如果激活的话，系统管理条形码导入网格的替代计量单位的列 |
| 位置 | 如果激活的话，系统管理条形码导入的位置列 |
| 客户/供应商账户 | 如果激活的话，系统管理属于条形码导入网格的账户、明细账目、客户/供应商说明的列 |
| 项目价格 | 如果激活的话，系统管理属于条形码导入网格的价格的列 |
| 选项 | 如果激活的话，系统管理条形码导入网格的选项列 |
| 变量 | 如果激活的话，系统管理条形码导入网格的变量列 |
| 拣货类型 | 表示拣货类型：项目类别/代码、批次、序号 |

#1.6 领货表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 营业额类型 | 系统将领货项目插入到发票的时候，本参数表示得利用的营业额类型 |
| 增值税 | 系统将领货项目插入到发票的时候，本参数表示得利用的增值税代码 |

#1.7 计价表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 根据模板计价额外组 | 在计价的过程中，表示货单分组的附加字段 |

#1.8 分析表格这些参数指定成本/利润中心的首先，然后将这些中心分配于发票项目。如果“重算”标记激活的话，通过选中的首先，用户可以插入/更新成本或者利润中心。
