title: 送货单参数
sidebar_position: 3

送货单参数使基础的设置正确管理，按照每个公司不同的需要。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 恢复参数 | 用于恢复初始参数 |

#1.1 总表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 对所有项目建议佣金 | 系统不再管理本参数 |
| 双重计量单位管理 | 如果激活的话，在货单中，用户可以管理替代的计量单位 |
| 自动建议替代计量单位 | 仅档“双重计量单位管理”参数激活，本参数可以工作。用户插入某个包含默认的替代计量单位的项目的时候，本计量单位自动地成为项目的替代计量单位。通过计量单位的缩放比例，系统计算替代数量 |
| 自动地建议承运人负责的运输 | 如果激活的话，承运的类型是运输，然后系统表示默认的运输 |
| 默认容量计量单位 | 如果激活的话，计算承运的时候作为容量的默认计量单位 |
| 默认重量计量单位 | 如果激活的话，计算承运的时候作为默认的重量计量单位 |
| 监控文档号和日期次序 | 如果激活的话，系统监控货单的日期/编号次序 |
| 仅在某一天中恢复 | 如果激活的话，在当前日期中，系统恢复可用的编号 |
| 自动恢复 | 如果激活的话，系统自动地建议第一可用的编号 |
| 监控可用性 | 如果激活的话，系统监控货单项目的可用性 |
| 排除销售订单 | 如果激活的话，货单项目的可用性计算不涉及销售订单的使用 |
| 必需的可用性 | 如果激活的话，货单项目的可用性是必需的，因此数量不能大雨可用 性 |
| 查看可用性 | 如果激活的话，系统将数量插入到行的时候，用可以查看项目的可用性 |
| 转包 | 如果激活的话，计算货单项目可用性的时候，系统涉及转包文档 |
| 采购 | 如果激活的话， 货单项目的可用性计算涉及采购文档 |
| 销售 | 如果激活的话，货单项目的可用性计算涉及销售文档 |
| 计划生产 | 如果激活的话，货单项目的可用性计算涉及物料需求文档和生产作业单 |
| 已发布的生产 | 如果激活的话，货单项目的可用性计算涉及生产订单 |
| 仓库 | 如果激活的话，货单项目的可用性计算涉及仓库记录 |
| 在所有预定义价目表中搜索项目价格 | 如果激活的话， 插入到货单行的项目，在客户登记中的价目表被搜索 |
| 用户代码为必须 | 如果激活的话，货单标头中的用户代码是必须的 |
| 使用条码 | 如果激活的话，系统管理属于项目网格的项目条形码的列 |
| 尺码和颜色管理 | 如果激活的话，系统管理变量网格的批次列 |
| 查看规模 | 如果激活的话，系统管理项目网格的规模列（高度、宽度、深度） |
| 重算运输 | 如果激活的话，系统自动地计算货单的运输 |
| 允许礼品费的折扣 | 如果激活的话，礼品管理折扣 |
| 监控完整性 | 如果激活的话，系统监控货单的监控完整性 |
| 外部订单参考 | 如果激活的话，系统管理与货单项目行有关的销售订单的外部参考 |
| 阻止日期为公众假日的文档插入 | 如果激活的话，系统阻止假日期的使用 |
| 最大行数量 | 指定项目网格中的最大数量 |

#1.2 履行表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 支付 | 在订单的多履行的情况下，表示  货单支付的利用价值：第一订单、登记或者手动选择 |
| 收件 | 在订单的多履行的情况下，表示货单收件的价值：第一订单、登记或者手动选择 |
| 监控货单支付款和订单一致性 | 如果激活的话，系统阻止包括不同支付类型的履行 |

#1.3 缺载表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 创建有文档日期的记录 | 如果激活的话，仓库记录包含送货单的日期，然后在缺载表格中，用户不需要指定记录路日期 |
| 优先级仓库和来自送货单中项目行的卸载模板 | 如果本参数激活的话，送货单行读仓库和缺载模板，如果不激活的话，系统利用已插入到本表格的仓库和模板 |
| 仓库 | 仓库首先和从送货单项目行缺载模板不激活的时候，表示货单缺载得利用的仓库 |
| 模板 | 仓库首先参数不激活的时候，表示货单缺载得利用的模板 |
| 自动装载/缺载 | 如果激活的话，“已打印”标记开始工作的时候，货单的缺载是自动的 |
| 通知虚拟项目 | 如果激活的话，缺载送货单的时候，系统监控如果送货单包含虚拟项目。如果有的话，某个问用户缺载送货单的信息出现 |

#1.4 导入表格在本表格中，为了将项目从条形码导入，用户指定文件成名和路线。为了打开导入表格，按照送货单管理的工具兰，在项目管理中。通过本导入操作用户可以将文件的项目插入到送货单文件。

#1.5 条形码导入的表格在本表格中，用户可以指定 为了导入属于BL_ImpBCAll表格的数据的参数。为了打开导入的表格，用户按照送货单管理的工具兰，在项目表格中。通过本导入的操作，用户可以插入送货单项目。



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 替代计量单位 | 如果激活的话，系统管理条形码导入网格的替代计量单位的列 |
| 位置 | 如果激活的话，系统管理条形码导入的位置列 |
| 客户/供应商账户 | 如果激活的话，系统管理  属于条形码导入网格的账户、明细账目、客户/供应商说明的列 |
| 项目价格 | 如果激活的话，系统管理属于条形码导入网格的价格的列 |
| 选项 | 如果激活的话，系统管理 条形码导入网格的选项列 |
| 变量 | 如果激活的话，系统管理条形码导入网格的变量列 |
| 拣货类型 | 表示拣货类型：项目类别/代码、批次、序号 |

#1.6 领货表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 营业额类型 | 系统将项目插入到送货单的时候，本参数表示得利用的营业额类型 |
| 增值税 | 系统将领货项目插入到送货单的时候，本参数表示得利用的增值税代码 |

#1.7 分析表格这些参数表示  成本/利润   中心的搜索首先，然后将这些中心分配于送货单项目。如果“重算”标记激活的话，通过选中的首先，用户可以插入/更新成本或者利润中心。
