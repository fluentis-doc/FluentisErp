---
title: 销售订单参数
sidebar_position: 2
---

销售订单参数使基础设置正确地管理，按照每个公司的需要。

功能导航栏：功能导航栏包含格式菜单，在这里可以执行相关过程的操作。包含的功能如下表：



| RibbonFunction | RibbonMeaning |
| --- | --- |
| 恢复参数 | 用于恢复初始的参数 |

#1.1 总表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 对所有项目建议佣金 | 系统不再管理本参数 |
| 双重计量单位单位管理 | 在销售订单中，替代计量单位被管理  |
| 自动建议替代计量单位 | 如果激活的话，在销售订单中替代计量单位被管理 |
| 覆盖编号的空缺 | 如果激活的话，系统恢复可用的编号 |
| 用户代码为必须 | 如果激活的话，在标头中指定用户是必须的操作 |
| 监控文档号和日期次序 | 如果激活的话，系统监控销售订单的日期/编号次序 |
| 仅在某一天中恢复 | 如果激活的话，只在当前中系统恢复可用的编号 |
| 自动恢复 | 自动地建议第一个可用的编号 |
| 自动插入变量到变量属性 | 如果激活的话，系统将属性变量自动地插入到登记 |
| 展开物料清单 | 如果激活的话，插入某个项目的时候，物料清单的第一个层级替代项目 |
| 监控可用性 | 如果激活的话，监控销售订单的可用性 |
| 排除销售订单 | 如果激活的话，销售订单项目的可用性的计算不包含销售订单的使用 |
| 必需可用性 | 如果激活的话，销售订单的项目的可用性为必需的，因此数量不能大于可用性 |
| 查看可用性 | 如果激活的话，插入数量的时候，系统表示项目行的可用行 |
| 转包 | 如果激活的话，销售订单的项目的可用性计算涉及转包文档 |
| 采购 | 如果激活的话，销售订单的项目的可用性计算涉及采购文档  |
| 销售 | 如果激活的话，销售订单的项目的可用性计算涉及销售文档 |
| 计划生产 | 如果激活的话，销售订单的项目的可用性计算涉及物料需求文档以及生产作业单 |
| 已发布的生产 | 如果激活的话，销售订单的项目的可用性计算涉及生产订单 |
| 仓库 | 如果激活的话，销售订单的项目的可用性计算设计仓库的记录 |
| 在所有预定义价目表中搜索项目价格 | 如果激活的话，插入到销售订单行的项目，在 客户登记和默认的价目表中被搜索 |
| 使用条码 | 如果激活的话，用户可以查看以及管理项目网格的客户项目和客户项目说明的列 |
| 使用客户项目 | 如果激活的话，用户可以查看以及管理项目网格的客户项目和客户项目说明的列 |
| 尺码和颜色管理 | 如果激活的话，用户可以管理变量网格的批次列 |
| 查看规模 | 如果激活的话，用户可以管理项目网格的规模列（高度、宽度、深度） |
| 允许礼品费的折扣 | 如果激活的话，礼品管理折扣 |
| 监控完整性 | 如果激活的话，系统监控销售订单数据的完整性 |
| 套装数量报告为必须 | 如果激活的话，系统监控  销售订单行的数量，关于项目登记的套装数量 |
| 阻止日期为公众假日的文档插入 | 如果激活的话，系统阻止假日期的使用 |
| 建议经济批次 | 如果激活的话，按照销售订单行的数量，系统建议项目登记的经济批次 |
| 关联采购/生产订单 | 如果激活的话，销售订单行的数量等于生产作业单的数量 |
| 合并属性 | 如果激活的话，系统以客户和项目登记属性为销售订单行的属性 |
| 阻止属性 | 如果激活的话，系统阻止销售订单的项目属性网格 |

#1.3 配置器表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 配置器 | 表示配置器的HTTP |
| 销售订单 | 表示HTTP WSOC |
| 版本 | 表示配置器的类型：配置器或者Visual Tailor |

#1.4 履行表格

| RibbonFunction | RibbonMeaning |
| --- | --- |
| 支付 | 在销售作业单的多履行的情况下，表示销售订单的支付的利用价值：第一订单或者手动选择 |
| 收件人 | 在销售作业单的多履行的情况下，表示收件人利用的价值：第一订单或者手动选择 |
| 从分组订单搜索的履行，按照支付的类型和解决方案 | 货单或者发票中的履行的时候，销售订单按支付的类型和解决方案分组 |
| 从分组订单搜索的履行 | 直接地履行的时候，在货单或者发票中，将销售订单按收件人和目的地分组 |

#1.5 分析表格用于指定成本中心/利润中心的优先：营业额类型、登记、项目登记或者仓库。
