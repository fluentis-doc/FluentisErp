| RibbonFunction | RibbonMeaning |
| --- | --- |
| 增值税账簿 | 对包括增值移动类型的模板是必须的。使用户将参考的增值税账簿分配于模板 |
| 增值税应计日期 | 通过本字段用户可以指定增值税移动的应记日期；在采购账簿中系统表示移动的记录日期，在销售账簿中表示文档日期。在欧洲经济共同体中增值税转账的模板的应记日期等于记录日期 |
| 文档类型/编号 | 本标记使文档编号的字段必须的，为了保存按本模板的记录 |
| 文档编号=增值税凭证   | 文档编号=增值税凭证  通过本标记，系统按照增值税凭证的顺序编号表示文档编号。文档的编号和凭证没有关系，因此字段不按照采购的记录。关于销售，我们建议用户将它应用于 在总会计中手动地完成的记录模板。对在销售发票类型设置的模板，我们建议用户不设置本标记 |
| 文档日期 | 如果本标记激活的话，保存按本模板的记录的文档日期的字段是必须的 |
| 增值税凭证=文档编号 | 增值税凭证=文档编号：通过本标记，系统  按照文档的编号，分配增值税凭证。因此，字段不按照采购记录，因为文档和凭证没有关系。关于销售，我们建议用户 将它  使用于销售发票类型，在发票会计中利用的。使用本设置的可能依赖于 得记录的所有文档发票类型的编号同一 |
| 打开交易 | 在使用本模板的会计记录中，本标记使创建账目的表格出现。本标记对所有的增值税记录以及对客户/供应商计划的管理很有用。对核查预支付的模板，用户可以设置本标记：在这个情况下，系统打开某个包括相反符号  的交易。核查下税文档的时候，通过交易系统得补偿 |
| 交易转移详细 | 通过本标记，系统将插入到属于交易的原来会计行的“转移详细”字段的文章插入到交易的票据。仅档上“”标记设置，本标记激活 |
| 打开每行运转 | 通过本标记，系统将某个交易插入到每个包括 某个明细账目转动行， 本明细账目包含管理打开的交易标记激活的。仅档“打开交易”标记激活，本标记也激活 |
| 交易关闭 | 在使用本模板的记录中，本标记使会计记录账目的关闭表格出现。普通地包括 不增值税的支付/收款 的所有记录。关于 信用记录  的模板以及包括某个交易打开与补偿的所有记录，它与交易打开和交易补偿一起被设置 |
| 使用历史汇率 | 仅档交易的关闭激活，通过本标记，按照初始的历史汇率，应用开始关闭的账目，未汇率差异的核查 |
| 在总账中的支付分组 | 通过本标记，仅档交易关闭激活可用的，应用将 与同一明细账目有关的交易关闭的所有总额分组于唯一的会计记录（尤其共享同一符号/货币的记录） |
| 从应计的佣金排除 | 通过本标记，仅档交易关闭激活可用的，本会计模板实现的交易关闭对供应商的佣金的计算无效 |
| 支付交易参考文档 | 通过本标记，仅档交易关闭激活可用的，在转动详细的说明中，应用开始管理 交易关闭的参考核查的（8）和（9）代码 |
| 交易报酬 | 通过本标记，仅档交易打开/关闭标记激活可用的，在按本模板的记录中，系统启动   补偿交易的过程。与信用记录文当有关的所有模板应该按照这些标记 |
| 警告应计期 | 保存会计记录的时候，本标记通知用户如果有得补偿的交易 |
| 交易分组 | 通过某个会计记录的核查的精确表格，标记启动交易分组的管理，不启动交易打开/关闭的标准管理 |
| 交易分组– 支付类型 | 用户将本支付类型分配于本模板核查的交易分组的移动 |
| 作业单 | 在会计记录中，本标记使 管理作业单计价的表格出现 |
| 成本中心 | 在会计记录中，本标记使管理成本计价的表格出现 |
| 利润中心 | 在会计记录中，本标记使管理利润中心的表格出现 |
| 机器管理 | 在会计的分析部分中，标记表示公司的车辆的成本/利润计价的网格 |
| 预付款的预扣 | 未利用的字段：无论本标记激活的话，会计支付的过程自动地核查预付款的预扣  |
| 固定资产 | 在会计记录中，本标记使管理固定资产部分的表格出现 |
| 操作类型 | 在会计的固定资产部分中的新移动的默认操作类型 |
| 更新余额 | 默认的标记 |
| 账簿移动的说明和支付 | 通过本标记过程将 记录的  普通说明复制于移动的详细说明以及支付票据 |
| 合同 | 识别合同中的记录，为税总结利用 |
| 货币 | 通过本标记，账簿部分表示货币字段 |
| 汇率/货币日期 | 通过本标记，账簿部分表示汇率以及货币日期的字段 |
| 货币金额 | 通过本标记，账簿部分表示货币金额字段 |
| 抵销明细账目 | 通过本标记，账簿部分表示抵销明细账目的字段。抵销明细账目字段的管理不建议的 |
| 转移详细 | 通过本标记，账簿部分表示转移详细的字段。我们建议用户对所有的会计模板设置本标记。通过这种方式用户一直可以插入会计移动的详细票据 |
| 部门 | 通过本标记，增值税和账簿部分表示部门字段。通过这种设置，用户可以文档分配于公司的部门 |
| 银行天数 | 通过本标记，账簿部分表示银行货币日期的字段。我们建议用户对核查财政移动的所有模板使用本字段。本字段的管理按银行登记中设置的货币天数（正/负）成为自动的 |
| 顺序编号 | 通过本标记，账簿部分表示行编号的字段。特别长的模板应该包括本标记。通过这种方式，用户按照模板可以排序记录 |
| 手动交易参考 | 通过本标记，账簿部分的交易的手动参考字段。字段将会计移动联系起来，交易的管理不再是必须的 |
