| RibbonFunction | RibbonMeaning |
| --- | --- |
| 自动模板 | 保存某个按模板的记录的时候，系统自动地生成其他的记录，按本字段表示的模板。为了正确地填写第二个记录，这两个记录得按照下列的记录逻辑：a) 在第一个情况下，第一个记录包括    在欧洲经济共同体中 采购的增值税移动类型，第二个包括在欧洲经济共同体中采购增值税转账。链接   依赖于  销售记录中的  采购增值税的核查。b)在第二个情况下，第一个记录打开某个交易，第二个记录关交易；c)第三个情况关于某个包含“公共实体销售增值税”移动类型的交易的 结账记录。本记录包含中断的增值税。在这个情况下，某个“中断增值税转账”记录与交易有关；d) 第四个情况，包括中断增值税的采购；f) 在第五情况下，第一个记录为增值税，第二个记录的价值按在会计科目表设置的自动明细账目的逻辑g) 如果上列情况不发生的话，系统按照第一个记录的增值税数据库设置第二个记录的数据。要不然，第二个记录表示  预期的明细账目，无价值 |
| 如果金额小于0，颠倒列 | 按照增值税移动的符号，系统可以颠倒贷方/债务的列（对所有的增值税记录，我们建议用户选择本字段） |
| 通知 | 仅档上字段激活的话，本字段也激活。系统通知用户符号已颠倒。本标记不影响会计的自动过程使用的模板 |
| 允许明细帐目值为0 | 通过本标记，过程：a) 使用户手动地插入包含金额为0的行b) 保存记录的时候，不删除模板表示的不包含价值的行c) 在账簿中，行给增值税债务/贷方价值。本标记意味着金额为0的删除，可是属于“增值税方式的率”表格的“在总帐中允许增值税为0”标记不激活 |
| 记录的原型 | 会计模板的基础（可是不必须的）部分。本部分包含  得管理的会计记录类型的普通模式。用户也可以插入普通的账户代码（例如供应商账目）。系统按照  总会计 参数包含的  客户/供应商账户的 清单，自动地更新这些代码。 这些明细账目以及债务/贷方部分影响会计的计价。手动地装载记录的时候，用户可以编辑这些会计基价。总额类型的代码值得精确的解释。它们按照增值税数据和销售帐给账簿某个价值。尤其：手动：记录的时候，用户手动地计价行；明细账目的应纳税：系统通过行的应纳税金额， 更新 每个属于增值税部分的明细账目的代码。对属于销售模板 的所有收益，我们建议用户使用这个金额的类型；应纳税总计：通过记录的应纳税总计系统更新行。系统不考虑账户/明细账目的代码；税额总计：通过记录的税额总计，系统更新行。系统不考虑账户/明细账目的代码。可扣除税：通过记录的可扣除税的总计， 系统更新行。系统不考虑账户/明细账目的代码；文档总计/记录：通过 记录的 明细账目的代码（如果它得按照会计参数的设置），系统更新行，按照文档/记录的总价值；应纳税+不可扣除税：为了更新行，系统按照记录的应纳税总计+不可扣除税。系统不考虑账户/明细账目的代码；应纳税+不可扣除税：为了更新行，系统按照记录的应纳税总计+不可扣除税。系统不考虑账户/明细账目的代码；50%应纳税：为了更新行，系统利用记录的50%应纳税。系统不考虑账户/明细账目的代码；50% 应纳税+不可扣除增值税：为了更新行，系统按照记录的50% 应纳税+增值税。系统不考虑账户/明细账目的代码；50% 应纳税+ 50%不可扣除增值税：为了更新行，系统使用记录的 应纳税总计的50%+不可扣除增值税的50%。系统不考虑账户/明细账目的代码；明细账目的应纳税+不可扣除税：通过明细账目的应纳税+不可扣除税的总额，对   每个属于增值税部分的明细账目代码，用户可以更新行。属于采购模板的所有成本应该按照这种总额的类型；盈利折旧：为了更新行，系统按照盈利折旧的总额；与交易关闭的模板有关；亏损折旧 ：为了更新行，系统按照亏损折旧的总额；与交易关闭的模板有关；明细账目的金额：为了更新行，系统使用属于记录的交易支付的总额，按照支付的符号；汇率差别的利润：为了更新行，系统 使用汇率差别的参考货币的利润的明细账目，按照汇率与 交易结帐的计价差异；如果汇率包含结帐标记激活，行不包括价值；汇率差异损失：为了更新行，系统使用 汇率差异的 参考货币的损失的明细账目，关于获得的金额，按照 汇率以及交易结帐汇率 的计价差异；如果汇率包含结帐标记，行不包括价值；自动的：被某些自动的会计过程使用的金额；某个抵销编码的字段（我们建议用户不使用本字段：会计已经包括某个表示抵消的报告，无论本设置）以及某个详细票据的字段完成网格。在本字段中，用户可以使用 掩码的下部分包含的所有代码，尤其对  属于  客户/供应商  账户   的交易结帐的模板，我们建议用户使用（8）或者（9）代码同时 “支付交易参考文档”标记（为了将已支付/已接收的参考记录于会计）。扣除税和增值税明细账目：为了计价增值税的行系统使用这个金额的类型。它包括增值税的可扣除的金额以及明细账目（按“增值税会计的类型”，与记录的登记或者“总会计参数”有关）。  |
| 会计记录的参数说明 | 在本部分中，用户可以将某个代码分配于标准说明，通过不同代码被完成。会计报告表示这些说明，特别在对帐单和账簿中。   对每个移动的行用户可以将某个代码分配于预设置的票据，与普通说明一样：仅本部分管理（8） 和（9）代码 |
