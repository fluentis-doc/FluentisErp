Datele din Controlling sunt gestionate întodeauna cu o arie de analiză asociată care identifică tipurile de date pe care le deține.

**Tipuri de arii**

*Buget*: reprezintă bugetul pe anul următor;  
*Situații finale*: pentru elaborarea închiderilor infranuale;  
*Multi-mediu*: arie fără valori (eventual cu măsuri fizice). De obicei conține regulile standard ale unui cost driver, reguli care pot fi specifice bugetului sau oricum diferite de procedurile pentru situațiile finale.

*Idea principală este că aria identifică setul de date care se elaborează/analizează*.

Când sunt elaborate datele pe arii, automat sunt șterse datele anului și sunt recreate cu situația din acel moment.

*Exemplu*: daca este elaborată situația finală pentru luna ianuarie, după o lună, când este elaborat ianuarie și febroarie, Fluentis elimină ianuarie și recalculeaza ianuarie și febroarie cu noile date deținute.

*FLUENTIS elimină lunile precedente și le recalculează împreună cu cele succesive*.
