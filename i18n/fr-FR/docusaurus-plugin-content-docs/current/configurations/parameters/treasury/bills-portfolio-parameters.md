RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Enregistrer | Bouton pour enregistrer les paramétrages. |
| Annuler paramétrages | Bouton pour annuler les paramétrages. |





Les paramètres du formulaire Portefeuille effets si trouvent en [Home > Paramètres > Trésorerie].

Dans les paramètres du formulaire Portefeuille effets est possible de préétablit les éléments par défaut à proposer dans les procédures du formulaire. En particulier:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Comptabilisation automatique | Il permet de prédéfinir la comptabilisation des effets directement au moment de l'acquisition effets par factures ou par comptes. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Cause de comptabilité | Il permet de définir la cause comptable par défaut à proposer dans la procédure homonyme. |
| Regrouper pour compte effet | Il permet de définir le paramètre de regroupement effet aux enregistrements effectués avec telle procédure. |
| Enregistrements provisoires | Il permet de définir l'état de l'enregistrement provisoire aux enregistrements effectués avec telle procédure. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Cause de comptabilité | Il permet de définir la cause comptable par défaut à proposer dans la procédure homonyme. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Cause de comptabilité | Il permet de définir la cause comptable par défaut à proposer dans la procédure homonyme. |
| Comptabiliser à la date d'échéance | La première option ira à casser l'enregistrement du registre aux diverses dates échéances des effets avec l'ajout des jours bancaires établir dans le registre banque pour la cause établie.  |
| Comptabiliser en date échéances avec date devise banque = date d'échéance | La deuxième option, au contraire, enregistrera les divers effets aux diverses dates échéances mais avec la date devise banque rectifiée selon les jours banques établis en registre banques pour la cause établie. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Cause de comptabilité | Il permet de définir la cause comptable par défaut à proposer dans la procédure homonyme. |
| Frais de commissions | Il permet de définir le sous-compte des frais de commissions pour les enregistrements effectués avec telle procédure. |
| Charges passives | Il permet de définir le sous-compte des charges passives pour les enregistrements effectués avec telle procédure. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Regrouper montants inférieurs à | Ce champ, si n'est pas vide et différence de zéro, détermine la valeur seuil des comptes à regrouper. Si, par exemple, il vient établi '1000', signifie que seront regroupés les comptes inférieurs à ce chiffre, donc jusqu'à 999,99.  |
