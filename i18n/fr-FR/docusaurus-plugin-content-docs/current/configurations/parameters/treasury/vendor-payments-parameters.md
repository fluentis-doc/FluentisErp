RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Enregistrer | Bouton pour enregistrer les paramètrages. |

 

Les paramètres du formulaire Paiements à fournisseur se trouvent en [Home > Paramètres > trésorerie].

Dans les paramètres du formulaire Paiement fournisseur est possible de préétablir les éléments par défaut à proposer dans les procédures du formulaire. En particulier:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Parcours de création des tracés reçu banc. | Paramétrage de la Directory standard de création du tracé télématique pour les virements Italie. On conseille de créer une Directory partage de réseau à l'intérieur d'un serveur, pour pouvoir accéder par tous les emplacements de la procédure. |
| Standard ABI | Paramétrage du format du fichier télématique.En ce cas on suggère le standard Abi, qui suive les paramétrages standard de communication ainsi comme définie par cet organisme. |
| Record HR | Paramétrage du formé du fichier télématique. |
| ABI Centre d'application | Paramétrage d'un code Abi de l'éventuel centre d'application qui avancera à véhiculer les communications (normalement est un champ ne nécessaire pas). |
| Imposer la banque d'appui présente dans le registre du fournisseur | Il est possible de définir l'utilisation de la banque d'appui relevée selon le défaut du registre relié: il se réfère d'écraser la banque d'appui insérée dans l'enregistrement comptable avec cette par défaut de la grille supérieur du registre banques. |
| Gestion réf. Iban | Paramétrage de l'obligation de gestion des codes Iban; en ce cas le tracé ne sera pas créé en cas ils manquent les ibans de certains paiements dans le registre. |
