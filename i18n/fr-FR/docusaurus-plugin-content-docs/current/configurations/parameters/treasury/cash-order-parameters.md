RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Enregistrer | Bouton pour enregistrer les paramétrages. |

 

Les paramètres du formulaire Reçu bancaire se trouvent en [Home > Paramètres > Trésorerie].

Dans les paramètres relatifs est possible de préétablir les éléments par défaut à proposer dans les procédures du formulaire. En particulier:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Parcours de création des tracés Reçu banc. | Paramétrage de la Directory standard de création du tracé télématique pour les reçus bancaires. On conseille de créer une Directory partage de réseau à l'intérieur d'un serveur, pour pouvoir accéder par tous les emplacements de la procédure. |
| Références du client | C'est la raison sociale de la société activée. |
| Description standard du crédit | Il est possible de coder, en exploitant les codes indiqués entre parenthèses, le texte à insérer dans le fichier quelle description du document de référence de l'émission reçue bancaire. Il se réfère à un texte du type Ft. n. (2) du (3). |
| Insérer caractère de contrôle final | Pour certains programmes de Remote Banking est nécessaire un retour au bout final dans le fichier, établir ce flag s'il est nécessaire. |
| Sans référence pour importation impayés | En certains programmes de Remote Banking est prévu une actualisation automatique des registres sur la base des données insérés dans le fichier reçu bancaire. Le champ qu'on utilise pour la procédure de paramétrages impayés peut causer une continue actualisation des registres qu'on peut éviter avec ce flag (ça comporte la n'utilisabilité pas de la procédure d'importation impayées). |
| Gestion échéances description | Avec ce flag la description du client insérée dans le fichier sera composée comme première description + deuxième description. |
