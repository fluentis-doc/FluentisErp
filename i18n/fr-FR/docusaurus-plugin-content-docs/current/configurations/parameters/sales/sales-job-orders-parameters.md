---
title: Paramètres commandes de vente
sidebar_position: 4
---

Les paramètres des commandes de vente permettent le paramétrage de base pour gérer correctement et selon les spécifiques demandes par chaque société.

RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Restaurer paramètres | Il permet de restaurer les paramètres aux valeurs initiales. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Exécution obligatoire | Il est utilisé dans les commandes clients: il signale que la commande client doit importer des articles par une ou plusieurs commandes de vente. |
| Exécution articles par commande obligatoire | Il est utilisé dans les commandes clients: il signale que la commande client doit contenir seulement des articles importés par la commande de vente. |
| Quantité fixe | Il est utilisé dans les commandes clients: il signale que la commande client ne peut pas exécuter par la commande de vente une quantité majeur de cette de la ligne article commande.  |
| Reprise des attributs par le registre articles | Il n'est pas géré. |
| Visualiser seul les articles relationnés | Il est utilisé dans le sous-traitant: il signale que dans l'exécution par la commande de vente est possible de faire seulement par les articles enfants. |
| Exécution commande en DDT livraison avec compte différent | Il est utilisé dans le sous-traitant: il signale que dans l'exécution de la commande de vente est possible de faire aussi par un compte différent de ce du bon de livraison de sous-traitant. |
| Gestion double unité de mesure | En ce champ il vient gérée l'unité de mesure alternative dans la commande de vente. |
| Proposition automatique UM alternative | En ce champ il vient proposée en automatique l'unité de mesure alternative de défaut. |
