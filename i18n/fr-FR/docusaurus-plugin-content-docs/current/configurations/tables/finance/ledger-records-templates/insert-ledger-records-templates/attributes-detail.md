RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Causale automatica | Au moment de l'enregistrement d'un enregistrement basé sur la cause qui est en modification le programme ira à créer automatiquement une deuxième enregistrement basée sur la cause indiquée en ce champ. Afin que le deuxième enregistrement soitcompilé correctement se doivent respecter les suivants logiques derelation entre les deux enregistrements:a) le premier cas est ce où le premier enregistrement est avec le type mouvement TVA achetés IntaCEE et la deuxième avec un virementTVA achetés Intra CEE, dans lequel la liaison est donnée par le relevé du TVA achats aussi dans le registre vente;b) le deuxième cas est ce où le premier enregistrement ouvre un compte au moment que la deuxième n'opère pas la clôture immédiate;c) le troisième cas est relatif à l'enregistrement de clôture d'un compte née avec le type mouvement ‘TVA vente à services publics' pour lequel le TVA est en suspension, cas où se relie un enregistrement de Virement TVA en suspension;d) le quatrième cas est l'achat avec TVA en suspension, au contraire du précédent;e)	le cinquième cas est ce où le premier enregistrement est TVA, au contraire, la deuxième est valorisée avec la logique des sous-comptes automatiques établi dans le plan des comptes;f) quand ne sont pas présents les cas précédents, le programme cherchera d'établir les données de du deuxième enregistrement sur la base des données TVA de la première; en alternative le deuxième enregistrement sera engendré en ramenant les sous-comptes prévussans aucune valeur établie.  |
| Inversion des colonnes si le montant est mineur de zéro | Il est possible de faire en façon que le programme inverse les colonnes Débit/Crédit d'entretien sur la base du signe du mouvement TVA (de ce champ on conseille la section pour tous les enregistrements TVA. |
| Avis | Le champ est actif seul si le précédent a été activé, consiste à revenir à l'utilisateur un message d'avertissement de l'inversion du signe. Le flag est insignifiant pour les causes utilisées dans les procédures automatiques de comptabilisation. |
| Permettre les sous-comptes avec valeurs à zéro | Avec ce flag la procédure:a) permettra d'insérer manuellement les lignes avec le montant soit débit que crédit égal à zéro;b) au moment de l'enregistrement de l'enregistrement N'iront PAS à supprimer les lignes ramenées par la cause et ne valorisées pas par l'utilisateur dans l'enregistrement;c) les lignes qui valorisent dans le libre journal le débit/crédit TVA ce flag comporteront de toute façon la supprimation du montant à zéro mais seul à condition que n'ait pas été inséré le flag 'Permettre TVA 0 en LJ' dans le tableau des 'Taux modalité TVA'. |
| Prototype de l'enregistrement | Cette est la section fondamentale (mais n'obligatoire pas) de la cause de comptabilité, dans laquelle va chargé le schéma typique du type d'enregistrement comptable à gérer. Est possible d'insérer aussi les codes génériques de compte (p.ex. le compte fournisseurs), qui seront actualisés automatiquement par le programme sur la base de la liste des comptes clients/fournisseurs insérés dans les Paramètres de comptabilité générale. Ces sous-comptes, ainsi comme les sections débit/crédit, guideront les valorisations comptables, qui seront de toute façon modifiable dans le cours du chargement manuel de l'enregistrement. Il faut expliquer en façon spécifique les codes de Type montant, fondamental pour valoriser automatiquement le livre journal sur la base des données TVA et du grand livre mouvementé dans l'enregistrement.En particulier:manuel: indique que la ligne sera valorisée manuellement par l'utilisateur à l'acte de l'enregistrement;imposable sous-compte: indique que la ligne sera actualisée pour chaque code de sous-compte inséré dans la section TVA avec le montant obtenu par l'imposable de la/les même/s ligne/es; l'utilisation de ce type montant est conseillée typiquement pour tous les produits insérés dans les causes de vente;total imposable: indique que la ligne sera actualisée avec le total imposable de l'enregistrement: aucune considération sera effectuée sur le code de compte/sous-compte inséré;total impôt: indique que la ligne sera actualisée avec le total impôt de l'enregistrement: aucune considération sera effectuée sur le code de compte/sous-compte inséré;impôt déductible: indique que la ligne sera actualisée avec le total de l'impôt déductible de l'enregistrement: aucune considération sera effectuée sur le code de compte/sous-compte inséré;le code du sous-compte titulaire de l'enregistrement (si conforme aux combinaisons insérées dans les Paramètres de comptabilité) pour la valeur totale du document/enregistrement;imposable impôt indéductible: indique que la ligne sera actualisée avec le total imposable sommé au total impôt déductible de l'enregistrement: aucune considération sera effectuée sur le code de compte/sous-compte inséré;50% imposable: indique que la ligne sera actualisée avec le 50% du total imposable de l'enregistrement: aucune considération sera effectuée sur le code de compte/sous-compte inséré;50% imposable TVA indéductible: indique que la ligne sera actualisée avec le 50% du total imposable de l'enregistrement sommé au total du TVA indéductible: aucune considération sera effectuée sur le code de compte/sous-compte inséré;50% impos. 50% TVA indéductible: indique que la ligne sera actualisée avec le 50% du total imposable de l'enregistrement sommé du 50% du TVA indéductible: aucune considération sera effectuée sur le code de compte/sous-compte inséré;imp.sous-c. impôt indéduc. sous-c: indique que la ligne sera actualisée pour chaque code de sous-compte inséré dans la section TVA avec le montant obtenu par la somme de l'imposable de la /des même/es ligne/es e du relatif impôt indéductible; l'utilisation de ce type montant est conseille typiquement pour tous les coûts insérés dans les causes d'achat;arrondissement actif: indique que la ligne sera actualisée avec le montant des arrondissements actifs; l'utilisation est à relier à cause de clôture des comptes;arrondissements passifs: indiquent que la ligne sera actualisée avec le montant des arrondissements passifs; l'utilisation est à se relier à cause de clôture des comptes;montant sous-compte: indique que la ligne sera actualisée avec le montant des paiements des comptes insérés dans l'enregistrement; selon le signe du même paiement;utile diff. changes: indique que la ligne sera actualisée avec le sous-compte d'utile de la devise de référence de la différence des changes, pour le montant obtenu selon le différentiel de valorisation entre le change historique et le change de clôture des comptes; en cas a été établi le flag de la clôture au change historique la ligne n'aura pas la valorisation aucune;pert .diff.canges: indiquent que la ligne sera actualisée avec le sous-compte de perte de la devise de référence de la différence des changes, pour le montant obtenu selon le différentiel de valorisation entre le change historique et le change de clôture du compte; en cas a été établi le flag de clôture au change historique la ligne n'aura pas valorisation aucune;automatique: type montant utilisé en certaines procédures de comptabilisation automatique.La grille se complète avec un champ de codification des contreparties (il en est déconseillé l'utilisation: en comptabilité est déjà présente une impression que visualisent les contreparties indipédamment par ce paramétrage) et par un champ de notes de détail: en ce champ sont possibles d'utiliser tous les codes ramenés à la base du masque, et en particulier pour les causes de clôture des comptes insérées en correspondance du compte clients/fournisseur générique on conseille d'utiliser les codes (8) ou (9) en concomitance avec le flag 'Réf. doc. compte en pai.' pour annoter en comptabilité les références des documents payés/encaissés.Impôt déduc. et sous-compte TVA: type montant utilisé pour valoriser la ligne relative au TVA avec le montant déductible du TVA et avec le sous-compte individué selon le 'Type comptabilisation TVA' associé en voix prioritaire au registre titulaire de l'enregistrement o par défault dans les 'Paramètres de comptabilité générale'. |
| Description paramétrique de l'enregistrement comptable | À l'intérieur de cet est possible de coder des descriptions standard qui vient rempliée par le programme à travers les divers codes indiqués, description qui seront ramenés dans les impressions comptables et en particulier dans les Relevés de compte et dans le livre journal. Aussi pour chaque ligne d'entretien est possible de coder des notes préétablies ainsi comme pour la description générique: en particulier les codes (8) et (9) sont gérés seuls en cette section spécifique de ligne. |
