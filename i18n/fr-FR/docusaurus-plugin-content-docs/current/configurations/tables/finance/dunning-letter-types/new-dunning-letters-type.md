| RibbonFunction | RibbonMeaning |
| --- | --- |
| Code | Code coordoné du type rappel. |
| Description | Description du type rappel. |
| Rappel pour le group des comptes | Le flag de gestion pour groupe de comptes, en particulier, ira à gérer une lettre de rappel pour un group de comptes au même temps (gestion standard), au contraire sans le flag de chaque compte comptable aura son rappel séparé par les autres. |
| Type intérêt | Optionnelle est la liaison au type de taux d'intérêt à appliquer au retard de paiement, pour obtenir un calcul à la date d'émission du rappel. |

Une fois enregistré le type de rappel il est possible de créer divers niveaux avec le bouton 'nouveau'. 



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Code | Code coordoné du rappel. |
| Description | Description du type rappel. |
| Priorité | Priorité de création de 1 à N niveaux. |
| Jours de retard | Indication des jours de retard qui doivent passer abord de solliciter le compte. |
| Jours | Le numéro de jours d'échéance du rappel et si ces soit à compter par la date de création ou par la date d'échéance du rappel précédent. |
| De date mouvement précédent | Date de référence pour les niveaux successifs au premier. |
| De date création | Date de référence normallement utilisée pour le premier niveau. |
| Crédit | La liaison à un type de contrôle crédit client n'est pas géré. |

Pour chaque numéro de rappel sera possible d'avancer, une fois enregistré, à l'insertion des divers textes en langue: le premier qu'on ira à changer est ce qui sera utilisé par défault au moment que le registre client n'a pas une langue assignée.

Les textes sont devises en 4 sections: l'objet de la lettre, l'en-tête, la conclusion et la section références (active seule pour les rappels sur les groupes des comptes). Dans la section références, en particulier, est possible insérer un texte en utilisant les codes prévus sur la base du masque: p.ex. 'Réf. (3) n. (4) du (5), (9) de (8) (10) avec échéances au (6)'.

RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Rechercher | Bouton pour rechercher le type rappel à travers les champs de filtre Code et Description. |
| Nouveau type | Bouton pour effectuer un nouveau type de rappel. |
| Modifier le type | Bouton pour modifier le type de rappel sur lequel s'est positionné. |
| Visualiser le type | Bouton pour visualiser le type de rappel sur lequel s'est positionné. |
| Supprimer le type | Bouton pour effectuer la supprimation du type de rappel sur lequel s'est positionné. |
