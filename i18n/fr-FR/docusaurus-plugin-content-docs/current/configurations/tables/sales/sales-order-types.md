---
title: Types commande client
sidebar_position: 19
---

Le type commande client est utilisé pour déterminer la propriété d'une commande client.

RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Rechercher | Bouton pour effectuer la rechercher d'un type document. |
| Insertion | Bouton pour insérer un nouveau type document. |
| Supprimation | Bouton pour supprimer le type document. |
| Aperçu avant impression | Bouton pour exécuter l'aperçu avant impression. |
| Impression | Bouton pour exécuter l'impression selon les filtres établis et la configuration de l'imprimante. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Types | Type commande client. |
| Description | Description du type ordre. |
| Type numér./description | Type numérotation associé au type ordre. |
| Auto commandes | S'il est établit, au moment qui vient inséré une commande client vient créée l'en-tête de la commande de vente avec le même numéro de la commande client et vient associée dans l'en-tête du même ordre. |
| Auto exécution | S'il est établi, l'ordre peut être exécuté par les documents DDT, prélèvement et factures de vente. |
| Bloquer le doc. impres. | S'il est établi, un ordre imprimé ne peut pas être modifié. |
| D-Exécution type bon de livraison | Il contient le type bon de livraison pour la création automatique DDT par le filtre commandes clients. |
| D-Exécution type facture | Il contient le type facture pour la création automatique factures de vente par le filtre commandes clients. |
| D-Exécution type prélèvement | Il contient le type Prélèvement pour la création automatique Prélèvement en exécution ordres disponibles ou en plans de charge. |
| Crédit | S'il est établi, l'ordre avec ve type ordre concurt au calcul du crédit. |
| Exploser le bloque | S'il est établi, l'ordre avec ce type ordre vient exclu par le bloque dérivé par le calcul du crédit. |
| Flux de trésorerie | S'il est établi, le type ordre concurt au calcul du flux trésorerie. |
| Entrepôt/Description entrepôt | Entrepôt d'engagement de la marchandise contenue dans les ordres de ce type ordre. |
| Cause/Description cause | Cause d'entretien de la marchandise engangée contenue en ordre de ce type ordre. |
| Sans additionner la quantité | S'il est établi, en exécution des ordres de ce type les quantités exécutées viennent ramenées dans le document d'exécution pour chaque ligne d'exécution. |
| Contrôle disponibilité | S'il est établi, les ordres de ce type concurrent au calcul de la disponibilité. |
| Agr.venteTVA | S'il est établi, il vient cherché pour l'article, s'existe, le code de TVA agricole; s'il n'existe pas il vient pris le code TVA normal. |
| Type comm. prod. | Dans définition MPS, la commande de production à commande client viendra créée du type spécifié en ce type. |
| Prix avec TVA | S'ils sont établis les prix des documents sont entendus avec TVA et par le prix vient calculée le TVA sur la base du code TVA et le prix sans TVA. |
| Type insertion | L'insertion automatique des articles en ordres peut avenir ou par le catalogue ou par le dernier ordre du même client. |
| Type commandes four./Description type CF | Le type commande fournisseur spécifié vient utilisé pour la création commandes fournisseurs par les commandes clients. |
| Contrôle client | Dans l'insertion du sous-compte du titulaire de l'ordre vient vérifier que le type compte est obligatoirement client. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Rapport d'impression | Il est possible de définir des rapports à utiliser pour la typpologie d'ordre. |
| Numéro de copies | Il est possible de définir le numéro de copies à imprimer pour la typologie d'ordre. |
