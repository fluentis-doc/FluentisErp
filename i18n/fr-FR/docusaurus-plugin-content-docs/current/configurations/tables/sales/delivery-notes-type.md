---
title: Type bons de livraison
sidebar_position: 13
---

Le type bons de livraison pour déterminer les propriétés d'un DDT.

RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Rechercher | Bouton pour effectuer la recherche d'un type document. |
| Insertion | Bouton pour insérer un nouveau type document. |
| Supprimation | Bouton pour supprimer le type document. |
| Aperçu avant impression | Bouton pour exécutr l'aperçu avant impresion. |
| Impression | Bouton pour exécuter l'impression selon les filtres établis et la configuration de l'imprimante. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Type | Type DDT. |
| Description | Description du type DDt. |
| Type n./Description | Type numérotation associé au type DDT. |
| Nature DDT | Il détermine le type DDt: livraison, retour et c/vision. |
| Ordres | S'il est établi, il signifie que le DDT peut être créé en exécutant une commande client. |
| Valeurs | S'il est établi, dans l'impression DDT peuvent être imprimés aussi les valeurs. |
| Type facture/Description | Type facture utilisé pour la création factures par DDT. |
| Causes trasp. | Description de la cause de transport. |
| Contrôle client | Dans l'insertion du sous-compte titulaire de l'ordre vient vérifié que le type compte soit obligatoirement client. |
| Bloque doc.impr. | S'il est établi, il n'est pas possible de modifier un DDT déjà imprimé. |
| Crédit | S'il est établi, le DDT avec ce type rentre dans le contrôle du crédit. |
| Excluses le bloque | S'il est établi, il signifie que aussi s'on supère le crédit, le document viendrait bloqué, est possible de toute façon l'insérer et l'imprimer. |
| Flux de trésorerie | S'il est établi, le DDT avec ce concurt au calcul du flux de trésorerie. |
| Entrepôt/Description | Entrepôt d'engagement de la marchandise contenue dans le DDT de ce type. |
| Cause/Description | Cause d'entrentien de la marchandise utilisée contenue dans le DDT de ce type. |
| Sans additionner quantité | S'il est établi, dans l'exécution des ordres de ce type les quantité exécutées viennent ramenées dans le document d'exécution pour chaque ligne d'exécution. |
| Agr.vente TVA | S'il est établi, il vient cherchée pour l'article s'existe le code TVA agricole, si n'est pas vient reprise le code TVA normal. |
| Contrôler le prélev. déch. | S'il est établi, au moment se crée un bon de livraison par un prélèvement décharge, s'active en automatique le flag déchargé pour le bon de livraison. |
| Bloquer qté PK | S'il est établi, bloque la quantité et la quantité alternative dans le bon de livraison créé par la liste UC. |
| Permettre le décharge lots ne disponibles pas | S'il est établi, il permet aussi le décharge des lots ne disponibles pas. |
| Gest.EDI | S'il est établi, pour le DDT de ce type est possible de créer le fichier EDI. |
| Prix avec TVA | S'il est établi, les prix présents dans le DDT sont compréhensifs de TVA. |



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Rapport d'impression | Il est possible de définir des rapports à utiliser pour la typologie d'ordre. |
| Numéro | Il est possible de définir le numéro de copies à imprimer pour typologie de DDT. |
