Le tableau 'Registres' est composé par 4 sections:

**Plan des comptes**: en cette section il faut indiquer, pour les sous-compte des registres bancaires, les champs 'Banque' et 'RBN'. Le premier indique l'institut de crédit de référence, au contraire le deuxième le type de compte (compte ordinaire, sbf....) associé au même sous-compte: ces deux champs sont visibles et modifiables aussi à l'intérieur du registre bancaire. Pour ce que concerne les sous-comptes de type client/fournisseur, en outre, est possible de gérer par ce masque aussi le champ du 'Dérogabilité', champ qui est possible d'gérer aussi par le masque principal du registre contacts, pour chaque ligne de sous-compte;

**Flux échéancier**: en cette section va attribue à chaque flux de trésorerie (il se réfère à la documentation de base du formulaire homonyme) le relatif code pour DocFinance. Il est permise l'attribution du même code sur plusieurs lignes flux;

**Types paiement**: en cette section va indiqué, pour chaque type de paiement, le relatif code de correspondance en DocFinance (on est permise l'attribution du même code sur plusieurs lignes). Il est possible d'indiquer un code de dérogabilité du type de paiement, qui sera utilisé dans l'exportation des données pour le DocFinance;

**Société**: pour chaque société est obligatoire indiquer le code correspondant pour le DocFinance. Les sociétés qui n'ont pas code seront exclues par les procédures d'exportation/importation des données pour/par le docFinance.

RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Rechercher | Avec ce bouton s'exécute la recherche en toutes les grilles du masque, selon les filtres établis. |
