Le tableau prévoit simplement la codification des typologies de flux de trésorerie que l'entreprise entend utiliser pour les simulations. Ils sont prévues les suivantes origines données:

**Comptes**: l'origine des données sont les comptes de comptabilité générale;

**Solde comptable**: à partir de ce type flux s'obtient les disponibilités financières initials, respect aux types comptes financiers codés;

**Commandes clients/commandes fournisseurs**: commandes clients, des typologies avec flag 'Flux de trésorerie', imprimés et confirmés, n'exécutés pas;

**Factures d'achat/vente**: factures imprimées/contrôlées mais pas encore comptabilisées (pas de type pro-forme);

**Extracomptable**: échéances manuelles extracomptables définies dans le formulaire Flux de trésorerie (voir le document successif);

**Receveurs**: rémunération des professionnels pas encore comptabilisés;

**Bons de livraison d'achat/vente**: bons de livraison de typologie avec le flag 'Flux de trésorerie', imprimés et confirmés, ne valorisés pas;

**Commandes et rentrées de sous-traitant**: commandes de sous-traitants à exécuter et rentrées de sous-traitants pas encore facturés;

**DA**: demandes d'achat;

**Ordres planifiés**: ordres de production avec phases extérieures planifiées ou ordres de production qui prévoient une demande d'achat.

On rappelle de coder de toute façon toutes les typologies prévues: la génération automatique permet de définir quels de ces considérer et quels pas.

RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Rechercher | Avec ce bouton s'exécute la recherche des types flux selon les filtres établis. |
| Nouveau type | Positionner le curseur à l'insertion d'une nouvelle ligne. |
| Supprimer les types | Supprimer le type flux sélectionné. |
