---
title: Matériaux utilisés
sidebar_position: 2
---

En ce tab ils viennent visualisés les matériaux que le sous-traitant devrait avoir utilisé pour produire ce qui a été inséré dans le tab 'Articles rentrées'. Le tab est formé par une en-tête, dont est possible de visualiser les informations de la ligne sélectionnée dans le tab 'Articles rentrées', d'une grille, dont viennent listés (avec la possibilité de modificaiton) les matériaux utilisés par le sous-traitant, par un autre grille, liée à la grille supérieur et dont est possible d'établir les lots des matériaux utilisés par le sous-traitant.

Les colonnes visualisées dans la grille sont les suivants:

**Classe, Code article, Variante, Description matériel et Description variante de l'article**: en ces colonnes viennent visualisées la classe, le code, la description article, la variante et la description variante du matérial utilisé par le sous-traitant;

**Option**: en cette colonne il vient visualisée l'option de la ligne article;

**UM**: en ce colonne il vient visualisée l'unité de mesure de gestion du matériel utilisé par le sous-traitant.

**Quantité utilisée**: en cette colonne il vient visualisée la quantité de matériel utilisée par le sous-traitant et qui devra donc être déchargée par l'entrepôt du sous-traitant. Cette quantité, éditable par l'utilisateur, vient proposée sur la base de la quantité unitaire établie dans le tab Matériaux à utilisée de la commande de sous-traitant, en correspondance de la ligne du même ordre exécuté avec la ligne de rentrée sélectionnée dans la tab Articles rentrées; cette quantité unitaire vient multipliée par la quantité rentrée et s'obtient donc la quantité utilisée;

**Quantité retour**: en cette colonne il vient visualisée la quantité de matériel que le sous-traitant rent et que dont devra être rechargée dans l'entrepôt retours;

**Entrepôt**: en cette colonne il vient visualisé le code de l'entrepôt retours, reprise par les paramètres de sous-traitant, visualisé automatiquement seul s'il vient insérée la quantité du retour dans la ligne du matériel, éditable par l'utilisateur;

**Cause**: en cette colonne il vient visualisé le code de la cause pour le chargé des retours, reprise par les paramètres de sous-traitant, visualisé automatiquement seul s'il vient insérée la quantité du retour du matériel, éditable par l'utilisateur;

**Description option, description entrepôt, description cause**: en ces colonnes ils viennent visualisées les descriptions d'option, entrepôt et cause établies dans la ligne du matériel.

Dans la grille en-dessous, l'utilisateur peut indiquer quelles sont les lots des matériaux qu'ont été visualisés par le sous-traitants. Les colonnes visualisées et le règles d'utilisation de la grille sont les mêmes respect à la grille des lots du tab Lots/NS qui si trouve dans le tab Articles rentrés.

RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Insérer | Il se réfère à la procédure pour insérer un nouveau rentrée: l'application enregistre automatiquement le rentrée à peine inséré et enleve tous les champs en rendant le fenêtre disponible pour l'insertion d'un nouveau rentrée. |
| Enregistrer | Bouton pour enregistrer les modifications effectuées dans le rentrée. |
| Rentrées prévues | Il se réfère à la procédure qui explode la fenêtre d'Exécution par l'ordre et qui permet de transférer par les commandes de sous-traitant les articles dans la rentrée. |
| Aperçu avant impression | Il visualise l'aperçu avant impression de la rentrée de sous-traitant. |
| Imprimer | Il lance l'impression du rapport de la rentrée de sous-traitant dans l'imprimante prédéfinie. |
| Documents | Il permet d'accéder à la Gestion de la documentation et de visualiser les éventuels documents joind à la rentrée de sous-traitant. |
