title: Calcul des règlements
sidebar_position: 2
---

Une fois exécutée la recherche, dans la section **Liste** ** agents**, seravisualisé tous les agents définis dans le système: en cette section il faut sélectionner la liste des sujets par laquelle exécuter le calcul. On remarque qui est aussi visible le type d'échéance de la commission de l'agent pour pouvoir établir les filtres de droit en façon apte aux paramétrages de l'agent.

Sur la droite il y a les suivants filtres:

- un filtre par devise du document, pour créer un règlement agent différencié par devise;

- trois sections de filtre selon le document de référence (ordre/bon de livraison/factures, par agents avec ordre confirmé, livraison effectuée, facture émise);

- une section de filtre sur les comptes, pour les agents avec échéances échues ou paiement passé: sur la base de la date établie seront donc, échus les comptes avec échéance antécédente dans le premier cas, dans le deuxième cas la quote-partie en relation aux encaissements en comptabilité avec date antécédente. Dans le cas des effets, ces ne seront pas considérés échus jusqu'à la date d'échéance (c'est-à-dire la date d'échéance plus les jours de risque client établis dans les paramètres du même formulaire).

Le flag 'Conversion des commissions en utilisant l'échange à la date document' prevoit au contraire que le montant des commissions soit effectué selon l'échange de la date du document, autrement sera utilisé par défaut l'échange de la date durèglement agent aussi qu'établis dans les champs sur la base de le masque.

Dans la section inférieure,'Extrêmes pour la création du règlementdes commissions', il va en fin indiquée la date du règlement, sa description et le mois/année de référence pour le calcul de l'Enasarco.

RIBBON BAR: il représente le menu de la forme en objet, à savoir la zone dans laquelle est possible d'effectuer des actions. La liste des fonctionnalités possibles est représentée de suite:



| RibbonFunction | RibbonMeaning |
| --- | --- |
| Rechercher | Le bouton execute la recherche des agents présents dans la société active. |
| Calcul des réglements | Avec ce bouton on exécute des réglements pour les agents sélectionnés, selon les paramétrages de filtre établis. |
